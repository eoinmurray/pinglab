## Introduction

Single-model companion to [nb010](/notebooks/nb010/). Trains only the [*ping*](/models/#ping) model — conductance-based synapses with the full E→I→E feedback loop that produces gamma oscillations — at *dt* = {cfg.dt ?? ## Method

Trained at *dt* = {cfg.dt ?? "?"} ms for *T* = {cfg.t_ms ?? "?"} ms per trial, on {cfg.max_samples ?? "?"} MNIST samples × {cfg.epochs ?? "?"} epochs. ## Next steps

The six shared training-health criteria pass — best acc {cell.best_acc ?? "?"}% is the highest of the five per-model entries at small tier, loss falls monotonically, hidden rate sits at {cell.final_acc ? "19" : "?"} Hz in band, no late-epoch collapse. But the seventh, *ping*-specific criterion **fails**: the inhibitory population is silent at the final epoch (*rate_i* = 0 Hz). That means the E→I→E feedback loop never closes — *ping* is achieving its 71% accuracy by training as feedforward-E (i.e. as *coba* with an unused inhibitory reservoir), not as PING. The trained weights no longer represent a PING circuit.

The likely cause is that *--ei-strength 0.5* is below the recruitment threshold for the small-tier input rate — nb004's ei-strength sweep showed I rates only rise at ei ≳ 0.3–0.4 with overdriven input, and this recipe runs at the default *input_rate_hz* without the 2× boost nb004 uses. Two candidate next moves: (a) bump *--ei-strength* to the canonical 0.8 from nb004 where PING is demonstrably formed, or (b) raise *--input-rate* so the E population drives I above its recruitment threshold. Either change should push *rate_i* off zero; the accuracy trade-off is open.
