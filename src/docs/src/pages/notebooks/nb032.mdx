## Introduction

[030 — COBA + NMDA cannot sustain](/notebooks/nb030/) showed the inhibitory loop is structurally required. [031](/notebooks/nb031/) located ping's sustained band on the $(W_{}, R)$ phase map. Earlier in this notebook (in a now-replaced run) I tried *binary match/non-match DMTS* with all three working pieces enabled — trainable $W_{}$, SGCC, NMDA on ping — and hit the same chance plateau the nb028/29 chain hit. Gradient flowed, but the optimiser fell into a uniform-output basin from which the per-sample gradient on the readout cancels across balanced batches.

That diagnosis pointed at the loss formulation, not the architecture. The LSNN paper's DMTS task isn't actually binary; it's **N-way store-and-recall** — present one of $N$ items, then later recall which it was. Multi-class cross-entropy has the per-sample directional variance that binary CE on balanced labels doesn't, and that's the same loss structure that let [nb027](/notebooks/nb027/) train COBA+SGCC to 87% on 10-class MNIST. This entry switches the task and re-runs.

## Method

1. **Sample** ({} ms) — one MNIST digit (class 0–9) Poisson-encoded at peak rate {} Hz across $N_{} = 784$ pixel channels.
2. **Delay** ({} ms) — input is identically zero. The network must hold class identity through this window via its own recurrent dynamics.
3. **Recall** ({} ms) — still zero input. The readout integrates E activity here.

**Network.** Identical architecture to the binary-CE attempt — ping COBANet (ei_strength = {}), $N_E$ = {}, $N_{} = 784$, *--trainable-w-ee* on, *--slow-syn* on at gain {}, *--sgcc* on at α = {}. $W_{}$ initialised at ({}, {}) μS (the low-edge regime from [nb031](/notebooks/nb031/), so the network can sustain but doesn't dump pattern info through gamma synchrony).

The point of integrating only over the recall window is to force the readout to use the *post-delay* state — it can't cheat by looking at the input directly. The post-delay E activity is purely recurrent dynamics, so what the readout sees has to be a function of what the network's been holding.

## Discussion

The architecture didn't change. Only the loss formulation did.

**Why the binary version failed and the N-way version doesn't.** With balanced match/non-match labels and an untrained hidden state that doesn't yet encode class identity, every match sample produces approximately the same hidden state as every non-match sample, so the per-sample gradient on the readout weights averages near-zero across the batch — the readout converges to uniform output, the loss locks at $\ln 2$, and gradient descent has no first-order incentive to escape. With multi-class CE on 10 distinct classes, even from a random hidden state each class produces a slightly different per-trial activity pattern (driven by different input pixels), so the per-sample gradient on the readout has *directional variance across classes*. The readout learns a class-discriminative projection from the start, and that gives $W_{}$ a non-trivial backprop signal to follow.

This is the same loss-structure that worked for nb027 (10-class MNIST classification on COBA+SGCC) and the same reason the LSNN paper's actual recall task is N-way rather than binary. The "DMTS is hard" finding from the nb028/29 chain was actually "binary CE on balanced labels is hard"; the architecture was always capable.

**What 66% means.** Chance is 10%, so this is 6.6× above chance — the network is clearly extracting class identity from the post-delay state. It's not yet competitive with feedforward MNIST classification (where the same COBA+SGCC stack hits 87% in [nb027](/notebooks/nb027/)) because:

1. The recall task only sees a 100 ms sample window (vs nb027's 200 ms full trial). Fewer input spikes → less reliable encoding.
2. There's a 200 ms silent delay between sample and readout, during which the network's hidden state has to maintain class identity by itself. Information is lost during the delay.
3. The readout integrates over 100 ms of post-delay activity which, depending on the operating regime, may be relatively quiet.

Headroom exists in all three dimensions (longer sample, more epochs/samples, larger network), but the *core question* of whether the architecture can hold class identity through a delay is now answered yes.

## Next steps

- **Longer delays.** Does test accuracy degrade gracefully as $T_d$ goes 200 → 400 → 800 ms, or does it cliff at the NMDA time constant ($\tau = 100$ ms)? This tests whether the recurrent dynamics ARE the slow store, or whether the network is just riding the NMDA tail.
- **Add ALIF on top.** ALIF ($\tau_{} = 700$ ms by default, configurable to 1200 ms à la LSNN) adds a much longer memory primitive than NMDA. Combined with NMDA the network has memory at two timescales; with ALIF only it's closer to LSNN architecturally.
- **Architectural changes, not more compute.** The large-tier run (n=8192 × 40 epochs, 101 min CPU) shows the curve plateaus at ≈63–66% from epoch 20 onward. The marginal accuracy gain compressed from +18 pp (small → medium) to +3.7 pp (medium → large), so more samples/epochs won't substantially help. To break past ≈70% the architecture needs to change: more E cells (currently $N_E = 256$), a longer slow primitive (ALIF $\tau_{} = 700$ ms on top of NMDA $\tau = 100$ ms), or a longer-integration readout (LI cells rather than mean-rate).
- **Try the binary version *with content selectivity already trained in*.** Warm-start from this notebook's trained weights and switch to binary match/non-match. The hidden state would start out class-discriminating, which should sidestep the chance-basin trap.
