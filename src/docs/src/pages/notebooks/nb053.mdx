## Abstract

The [Training](/articles/ar006/) article proves that the gradient explosion in conductance-based spiking networks comes specifically from the recurrent E→I→E loop: the backpropagated voltage gradient compounds once per gamma cycle by a loop gain $\varrho \propto W_{ei}W_{ie}$, and the `--v-grad-dampen` flag exists to tame it. That predicts a clean falsifiable consequence — **if the loop is the cause, then a network with the loop disabled should not need the flag**. This entry tests it directly. A network run as COBA (ei-strength 0, loop off) and as PING (ei-strength 1, loop on), each trained with damping off ($\gamma = 1$) and on ($\gamma = 1000$), otherwise identical. The prediction holds: COBA trains to {fmtA(C["coba__damp_off"]?.best_acc)}% with damping completely off, while PING with damping off has *every* optimiser step's gradient go non-finite and freezes at chance.

## Methods

Four MNIST runs on a 2×2 grid — {`{coba, ping}`} × {`{γ = 1 (off), γ = 1000 (on)}`} — using the canonical recipes from [nb025](/notebooks/nb025/) verbatim, with only `--v-grad-dampen` varied. Both models build as the same COBANet class; COBA simply sets `--ei-strength 0`, which zeroes the E→I weight $W_{ei}$ and so breaks the loop (the I population receives no drive and stays silent). All knobs are hardcoded literals in *src/notebooks/nb053.py* per the [runner contract](/docs/the-notebooks/#runner-contract).

The decisive measurement is the **number of skipped optimiser steps**. The trainer computes the pre-clip gradient norm each batch and, if it is non-finite (Inf/NaN), skips the step — so the skip count is a direct, unambiguous readout of the explosion. We also track final test accuracy and the peak finite pre-clip gradient norm.

## Discussion

The result confirms the [ar006](/articles/ar006/) mechanism by elimination. Damping is not a generic numerical safeguard for conductance-based networks — it is load-bearing exactly when a recurrent excitatory–inhibitory loop is present, and inert without one. COBA carries the same conductance dynamics, the same surrogate, the same membrane stiffness as PING; the only thing it lacks is the loop, and that single difference is the difference between needing the stabiliser and not. The loop gain $\varrho \propto W_{ei}W_{ie}$ vanishes when $W_{ei} = 0$, so there is nothing to compound, and the leftover feedforward gradient is bounded.

A practical corollary: the canonical recipes apply $\gamma = 1000$ uniformly to the whole conductance family, including COBA. For COBA that is conservative rather than necessary — harmless, but per the [ar006 corollary](/articles/ar006/) it does divide the feedforward learning signal by $\gamma$, which is why COBA-off here reaches essentially the same accuracy with a much larger, un-shrunk gradient.

## Success criteria

- COBA trains with damping off (skips zero steps, accuracy well above chance): **met** — {fmtA(C["coba__damp_off"]?.best_acc)}%, 0 skipped.
- PING fails with damping off (steps skipped, accuracy at chance): **met** — {C["ping__damp_off"]?.skipped_steps} skipped, {fmtA(C["ping__damp_off"]?.best_acc)}%.
- Damping restores PING: **met** — {fmtA(C["ping__damp_on"]?.best_acc)}%.

## Next steps

Re-enabling recurrent excitation (a non-zero $W_{ee}$) in an otherwise-COBA network should reintroduce a loop — and therefore the need for damping — even with the inhibitory loop still off, isolating the loop (rather than inhibition specifically) as the cause. Sweeping ei-strength from 0 upward would locate the threshold where the loop gain crosses 1 and damping becomes mandatory.
