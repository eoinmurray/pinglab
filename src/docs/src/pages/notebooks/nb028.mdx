## Introduction

The DMTS chain so far:

- [nb025](/notebooks/nb025/) added a slow ("NMDA-like") synaptic channel and tried to train DMTS on top of it. Loss stayed at chance regardless of LR or trial schedule.
- [nb026](/notebooks/nb026/) added ALIF (LSNN-style per-neuron adaptation), same outcome. Also mapped the *--v-grad-dampen* / LR space deliberately — found no Goldilocks setting and concluded the limiter is the gradient *stabiliser*, not the memory mechanism.
- [nb027](/notebooks/nb027/) replaced *--v-grad-dampen 1000* with SGCC on coba MNIST and showed the principled controller matches the baseline through epoch 10 and pulls 3.5 pp ahead with more training. Confirmed empirically that the global dampener was capping gradient signal.

This entry runs the actual experiment those three were building toward: **coba + ALIF (memory) + SGCC (gradient stabiliser), trained from scratch on DMTS.** Same task as nb026 ($T_s = T_d = T_p = 50$ ms, binary match/non-match), same hyperparameters (lr = 4×10⁻⁴, batch = 256), only the gradient stabilisation has changed.

If the diagnosis from nb026 + nb027 is correct, this should escape the chance plateau. If it doesn't, the bottleneck lies further up the stack.

## Method

Network is COBANet with `ei_strength=0` (coba), `--alif` (β = 1.7 mV/spike, τ_a = 700 ms), `--sgcc` (α = 0.5), `mem-mean` readout, `N_OUT = 2`. Same MNIST encoding as the prior DMTS attempts: 25 Hz Poisson during sample/probe phases, silent during delay.

The crucial swap relative to nb026: *--v-grad-dampen* is NOT set (left at its default of 1.0 = off). SGCC handles the conductance Jacobian explosion surgically — scaling only the voltage↔conductance cross-coupling gradient by α — so the diagonal gradient pathway that the global dampener was muting stays intact.

## Discussion

The DMTS training landed in the third category: stuck at chance, no NaN at delay = 0. We can now finally rule out two whole hypotheses and isolate the last one:

1. **Gradient stability** ✓ solved. With SGCC at α = 0.2 there is no NaN through the first ~5 epochs of curriculum delay = 0, and the optimizer takes finite-gradient steps. Without SGCC and at the same recipe ([nb026](/notebooks/nb026/)) we would have NaN'd by now.
2. **Memory mechanism** ✓ adequate. ALIF provides the long timescale, and curriculum step 1 (delay = 0) doesn't even require memory — yet still doesn't learn.
3. **Cold-start binary credit assignment** ✗ unsolved. With balanced 50/50 match/non-match batches and untrained readout weights, the 2-class cross-entropy gradient cancels across the batch: half the samples push (logit_match − logit_nonmatch) up, half push it down, mean zero. Until the network's hidden representation actually encodes sample-vs-probe matching, there's no signal pointing the optimizer anywhere — and at cold start, the hidden state has no reason to encode that.

The curriculum doesn't escape this because *every* stage is a 2-class task. Stretching the delay from 0 ms only changes how hard the memory part is, not the binary credit assignment.

## Why this isn't a dead end

The curriculum result actually narrows the next move precisely. nb027 demonstrated that coba+SGCC trains cleanly on a *10-class* MNIST objective and ends at 87% test accuracy — i.e. the network *does* have a usable class-discriminating hidden representation when the task is multi-class. The right experiment now is **warm-start**: load nb027's *coba_sgcc__seed42* weights, swap the 10-class readout for a 2-class match/non-match head, and continue training on DMTS. The hidden representation already encodes digit identity; the gradient just has to find a binary projection from features that already discriminate.

## Next steps

- **Warm-start from nb027 weights** (the obvious next experiment, separate notebook).
- **Alternatively, contrastive / triplet loss** instead of binary CE — produces non-cancelling per-sample gradient even at chance plateau.
- **Multi-way match task** (e.g. 4-way: which of 4 candidates matches the sample?) — multi-class loss has the same gradient-direction-diversity that made nb027 work.
