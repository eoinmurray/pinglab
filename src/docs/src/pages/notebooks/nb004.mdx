## Introduction

Stub entry for the [Spiking Heidelberg Digits](https://zenkelab.org/resources/spiking-heidelberg-datasets-shd/) dataset, introduced by Cramer et al. ([*cramer-et-al*](/references/#spiking-mnist--benchmarks)) for systematic SNN benchmarking — 20 classes of spoken digits (English + German) encoded as cochleogram spike trains over 700 input channels and $\sim$1 s per trial. SHD is natively event-based, so the dt / discretisation / encoding questions the earlier notebooks studied apply directly rather than being mediated by an artificial row-by-row encoding of an image dataset.

### Comparison to the literature

Cramer et al. ([*cramer-et-al*](/references/#spiking-mnist--benchmarks)) set the reference SHD recipe that most subsequent work measures against: a current-based LIF with exponential synapse, LI (non-spiking) readout, full-BPTT with a fast-sigmoid surrogate, and a two-sided firing-rate regulariser. Their headline numbers are 48.1–48.6% for 1–2 layer feedforward SNNs and 71.4% for the recurrent variant at matched capacity.

This entry deliberately tracks that recipe component-by-component so the gap to the reference number is interpretable. Below are the features of their setup and whether this entry matches.

**Matching**

| Feature | Cramer et al. | This entry |
|---|---|---|
| Neuron | current-based LIF (cuba) | *cuba-exp* |
| Synapse | exponential low-pass on spikes | exponential low-pass on spikes |
| $\tau_\text{mem}$ | 20 ms | 20 ms |
| $\tau_\text{syn}$ | 10 ms | 10 ms |
| Trial length $T$ | 1 s | {cfg.t_ms ?? "?"} ms |
| Timestep $dt$ | 2 ms | {cfg.dt ?? "?"} ms |
| Hidden width | 128 | 128 |
| Readout | LI (non-spiking leaky integrator) | LI |
| Input channels | 700 cochleogram | 700 cochleogram |
| Batch size | 256 | 256 |
| Learning rate | $10^{-3}$ | $10^{-3}$ |
| Surrogate | fast-sigmoid | fast-sigmoid |
| Firing-rate regulariser | two-sided, $\theta_l{=}0.01$, $s_l{=}1.0$, $\theta_u{=}100$, $s_u{=}0.06$ | same |
| BPTT | full | full |

**Not matching**

| Feature | Cramer et al. | This entry | Notes |
|---|---|---|---|
| Architecture | recurrent (hidden→hidden) for the 71.4% row, feedforward for the 48% rows | feedforward only (*W_rec* = 0) | we're targeting the 48% feedforward band first; recurrence is the next step |
| Hidden depth | 1 layer for the 48.1% row, 2 layers for 48.6% | 1 layer | width $\times$ depth budget not yet swept |
| Epochs | 150 | {cfg.epochs ?? "?"} | our **{numbers.tier ?? "?"}** tier is much shorter for iteration speed |
| Training samples | full SHD (8156 train) | {cfg.max_samples ?? "?"} | subsampled per tier; *huge* tier will restore full-set |
| Surrogate slope $\beta$ | 40 | 10 | $\beta{=}40$ × 500-step BPTT explodes gradients in our current config; we'll revisit once a stable backbone is established |
| Optimizer | Adamax | Adam | not yet wired as an oscilloscope flag |
| Weight init | Kaiming uniform, $k{=}1/\text{fan-in}$ | Kaiming uniform, $k{=}2/\text{fan-in}$ (PyTorch default) | factor-2 difference in init scale |
| Data augmentation | none for the 71.4% row; time-jitter/channel-shift for 83.2% | none | out of scope until the recurrent-SNN number is reproduced |

## Success criteria

- **SHD pipeline trains end-to-end** on 700-channel cochleogram input at the **{numbers.tier ?? "?"}** tier without NaN / collapse.
- **Above chance**. Test accuracy materially above the 5% floor (20 classes).
- **Matches the feedforward-SNN reference**. Best accuracy lands inside Cramer et al.'s 47.5–48.6% feedforward-SNN band.

## Next steps

**Align the cuba-exp update with Zenke's canonical SHD form.** The canonical SHD baseline (Zenke's Spytorch Tutorial 4, which Cramer et al. 2022 cite) uses

$$
\text{syn}_{t+1} = \alpha\,\text{syn}_t + W\!\cdot z_t, \qquad \text{mem}_{t+1} = \beta\,\text{mem}_t + \text{syn}_{t+1}
$$

with a bare weight init $W \sim \mathcal{N}(0, 0.2/\sqrt{N_\text{in}})$ — no $(1-\alpha)$ or $(1-\beta)$ factor on the drive. Our cuba-exp uses the dt-invariant low-pass form $\beta\,\text{mem} + (1-\beta)\,\text{drive}$, which forces a compensation multiplier on the init ($isw$, $isb$) to match Zenke's per-spike kick. The compensation is currently tuned to 6 × Kaiming, empirically sitting in the stable-but-undermatched middle; attempts to push slope past 5 blow up in ways Zenke's form does not.

The cleanest path to the Cramer 47–49% feedforward band — and the only way to run at $\beta = 40$ like Cramer — is to match Zenke's update *form* directly, not to keep bisecting $isw$. Concretely: add a `--cuba-update-form {low-pass, direct}` flag on the oscilloscope, use `direct` ($\beta\,\text{mem} + \text{drive}$, $\alpha\,\text{syn} + W\!\cdot z$) for nb004, and initialise $W$ with weight_scale = 0.2 directly — not the Kaiming × compensation stack.

**Re-enable recurrence.** The 71% RSNN target requires $W_\text{rec} \neq 0$. Feedforward $W_\text{rec} = 0$ with slope = 5 + adamax is stable; under the direct update form above, restoring $W_\text{rec} \sim \mathcal{N}(0, 0.2/\sqrt{N_\text{hidden}})$ should match Zenke's `v1`.

**Remove the grad-skip band-aid.** Adamax reduces skipped batches from ~440 to ~3 across a 69-epoch run; the `--skip-bad-grad-threshold 1e4` flag is no longer load-bearing. Already dropped from the recipe.
