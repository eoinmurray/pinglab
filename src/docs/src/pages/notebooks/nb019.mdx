## Introduction

Train a [*coba*](/docs/models/#coba) network — COBANet dispatched through the *ping* arch with *ei_strength* = 0, so the E→I→E loop is silent during BPTT — then run inference on the same held-out test set across an *ei_strength* sweep with the same trained weights, measuring how classification accuracy and hidden firing rates degrade as the inhibitory loop the network never saw during learning is engaged at progressively higher strength. The question is whether the readout depends on a population code the I-loop merely modulates, or on rate / timing features that the inhibitory shunt destroys.

The shared-arch trick is what makes this a one-knob comparison: *coba* and *ping* are the same COBANet — the same $W_\text{in}$, $W_\text{rec}$ (E→E), $W_\text{ei}$, $W_\text{ie}$, $W_\text{out}$ — and *ei_strength* is just a scalar on the loop weights. Trained-coba weights load 1:1 into the ping graph, no shape mismatch, no projection.

## Method

Every hyperparameter is a hardcoded literal in *src/pinglab/notebooks/nb019.py* per the [runner contract](/docs/the-notebooks/#runner-contract); the runner shells out to [oscilloscope *train*](/docs/the-oscilloscope/#subcommand-train) once and then runs the inference sweep in-process. The training recipe mirrors [nb011](/notebooks/nb011/) exactly. Each sweep point builds a fresh ping net at its own *ei_strength* and selectively loads from the trained checkpoint — *W_ff* (input → hidden, hidden → output) and *W_ee* (recurrent E→E) are transferred; *W_ei* and *W_ie* (the inhibitory loop weights) are skipped and kept at their fresh init.

- **ei_strength = 0 at train** — collapses the COBANet dispatch into a pure conductance-based feedforward path by zeroing the E→I→E loop. The trained network has never integrated an I-population spike on the way to its loss.
- **ei_strength sweep at infer** — the inference grid covers 0 → 1.0 in 0.1 steps. The bottom is identity (coba native); the top is the upper end of the canonical PING-on regime ([nb013](/notebooks/nb013/) ping recipe).
- **selective state-dict load** — *W_ei* and *W_ie* in the trained checkpoint are zero matrices, because *ei_strength* = 0 at training scales their init by zero, and they carry no gradient (*requires_grad* = False) so they stay zero throughout. Loading them would silently nullify the I-loop at any inference *ei_strength*. Skipping those keys lets the freshly-built I-loop survive into each sweep pass.
- **same eval seed and same test split across the sweep** — every sweep point uses the canonical 80/20 split and the deterministic eval-time Poisson generator, so the only thing varying across rows is the inhibitory loop strength.

## Discussion

Figure 3 is the mechanistic story. The trained-coba network never integrated an I-population spike during BPTT, so the readout is locked to whatever feature the asynchronous coba E rasters expose. Engaging the I-loop reshapes those rasters into a fundamentally different population code — synchronous gamma bursts rather than dense async firing — and yet the readout extracts $\sim$74% on MNIST out of the burst-train regime, only $\sim$10 pp below baseline. That means the feature it locked onto is preserved across both regimes: it's not gross E rate (which drops 10×) and it's not the fine timing of individual coba spikes (which the I-loop demolishes). It has to be something coarser — likely the *spatial* pattern of which hidden units fire at all, scaled up or down in rate by the inhibitory shunt. The downstream LIF readout is integrating over the whole 200 ms window, so it sees a time-averaged population signature that's largely conserved.

The accuracy curve in Figure 1 has a soft knee at ei ≈ 0.4–0.5 that lines up with where Figure 3's raster cycles become tight and regular — once PING is established, further increases in *ei* don't restructure the code, just sharpen its modulation depth. Below the I-onset threshold (ei $\lesssim$ 0.15) there's no degradation at all because the I population is silent (Figure 2).

## Next steps

Symmetric direction: train ping at *ei* = 1 and infer at *ei* = 0, to see whether removing the loop is the harder direction. If the transfer is asymmetric, that asymmetry is itself informative about which dynamical features the gradient locks onto.
