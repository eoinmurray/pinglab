## Abstract

**PING** — Pyramidal-Interneuron Network Gamma — is a clock built from two populations: excitatory (**E**) cells fire, recruit inhibitory (**I**) cells, the I volley silences E, the GABA conductance decays, and E fires again. The cycle repeats at ≈30–80 Hz, but only when E gets enough external drive; below threshold the network sits in sparse, asynchronous background firing. A three-phase drive — low → high → low input — switches the recurrent loop on and back off so you watch the gamma clock ignite and extinguish in real time.

## Method

One continuous {fmtHz(3 * (cfg.phase_ms ?? 0) / 1000)} s forward pass on a single fixed PING network — no scan, no reset between phases. The {cfg.n_e}-cell E population is laid out on a {cfg.grid}×{cfg.grid} grid; the drive is a uniform Poisson input over {cfg.n_in} channels whose per-channel rate follows three equal {cfg.phase_ms} ms windows:

- **async (low)** — {fmtHz(cfg.low_rate_hz)} Hz per channel: sub-threshold, the network idles in sparse background firing.
- **PING (high)** — {fmtHz(cfg.high_rate_hz)} Hz per channel: supra-threshold, the E↔I loop locks into gamma volleys.
- **async (low)** — back to {fmtHz(cfg.low_rate_hz)} Hz: the clock releases and the grid returns to sparse firing.
