## Introduction

[nb024](/notebooks/nb024/) trains COBA on MNIST to $\sim 84$% test accuracy only with *--v-grad-dampen 1000* — a heavy-handed global gradient scalar that hides the conductance-Jacobian explosion by muting every per-step voltage gradient uniformly. The cost of that stabilizer is that the same uniform damping which prevents NaN also kills the gradient signal needed for any task longer than a couple hundred milliseconds — so it's unusable for working-memory work without something more surgical.

Burghi, Pugliese Carratelli & Rule (2024) — *"Costate control for nonlinear system identification with applications to excitable systems"* — propose a principled alternative for conductance-based neuron models: **Surrogate Gradients by Costate Control (SGCC)**. Instead of damping every voltage gradient uniformly, SGCC inserts a control input into the costate (adjoint) dynamics that targets *only* the cross-coupling terms between voltage and conductance — the path responsible for the explosion. Two designs:

1. **Proportional high-gain** $K^\text{prop} = -\gamma I$: provably eliminates the explosion but smooths the gradient landscape uniformly. nb024's *--v-grad-dampen* is essentially a discretised $K^\text{prop}$ with very large $\gamma$.
2. **Gating-based** $K^\text{gat}$: surgically suppresses only the destabilising off-diagonal terms in $\nabla_x f_\theta$ by a factor $(1-\alpha)$. Preserves more landscape structure, so gradient descent retains more signal.

This entry adds $K^\text{gat}$ to pinglab as the *--sgcc / --sgcc-alpha* CLI flag pair, then trains a fresh COBA on MNIST with *--sgcc-alpha 0.5* (and no *--v-grad-dampen*). Same recipe as nb024's COBA baseline in every other respect — the only difference is the gradient stabilizer. The question is whether the principled controller matches or beats the heavy-handed one on the simplest established task.

## Method

The K^gat controller is implemented via PyTorch's autograd. For each timestep, the per-layer conductance $g_e$ (and $g_i$ when an inhibitory population is present) is wrapped in a function that's identity on the forward pass but scales the gradient by $\alpha$ on the backward pass:

$$
g_e^\text{step} = \mathrm{ScaleGrad}(g_e, \alpha), \qquad v^\text{new}, s^\text{new} = \mathrm{LIF\,step}(v, g_e^\text{step}, g_i^\text{step})
$$

Forward dynamics are bit-identical to plain COBA. Backward, gradient flowing from $v^\text{new}$ to $g_e$ is multiplied by $\alpha$ — exactly the off-diagonal block the paper's $K^\text{gat}$ targets, without touching the diagonal $\partial g_e^\text{new}/\partial g_e^\text{old}$ or the gradient to upstream parameters except through this single multiplicative.

The single $\mathrm{ScaleGrad}$ is implemented via the existing *_scale_grad* helper. Parity at $\alpha = 1$ verified bit-exact; gradient norm drops $\sim 6 \times$ at $\alpha = 0.5$ on a smoke test.

**Training recipe**: identical to nb024 COBA (*--ei-strength 0*, *--w-in 0.3*, *--w-in-sparsity 0.95*, *--readout mem-mean*, *--readout-w-out-scale 100*, *--lr 0.0004*, *--batch-size 256*), with *--v-grad-dampen 1000* replaced by *--sgcc --sgcc-alpha 0.5*. Small tier: 500 samples × 5 epochs.

## Discussion

SGCC works as a drop-in replacement for *--v-grad-dampen 1000*, and on the same recipe with 2× more epochs it pulls 3.5 percentage points ahead.

Through the first 10 epochs (where the nb024 baseline run terminates), the SGCC and baseline loss curves are essentially indistinguishable in Figure 1 — both descend from 2.3 to ~0.55, both reach $\sim$84% test accuracy. There is no NaN, no chance plateau, no obvious training pathology. The principled $K^\text{gat}$ controller delivers exactly the kind of gradient signal *--v-grad-dampen 1000* was simulating with a much heavier hand.

The interesting half of the figure is epochs 10–20. The baseline run ended at epoch 10 (nb024's tier choice); the SGCC run continued. Loss kept descending (0.57 → 0.40); test accuracy climbed from 84.5% → 87.75%. The implicit message is that *--v-grad-dampen 1000* has been *capping the achievable accuracy* via its uniform gradient suppression — there's nothing wrong with the network or the optimizer or the task at epoch 10; it's just that the global dampener silently throttles further progress. SGCC, which preserves the gradient direction perpendicular to the explosion axis, retains useful signal that the dampener was washing out.

This is exactly the result Burghi et al. predict in their proposition 1: the original gradients and the SGCC surrogate gradients vanish at the same global optimum, but in between, the SGCC variant retains significantly more landscape structure. The trained-network rasters in Figure 2 corroborate the dynamical equivalence — same operating point ($\sim$85 Hz E rate, asynchronous-dense activity, silent I population because *ei-strength = 0*), reached via a different optimization path.

**Implications for the harder tasks.** SGCC is now an empirically validated drop-in for the *--v-grad-dampen* stabilizer on MNIST, with strictly more landscape structure preserved. That makes it the right primitive to reach for on any task where the global dampener was capping signal — working-memory loops being the obvious case.

## Next steps

- **Sweep α** across e.g. {0.3, 0.5, 0.7, 0.9} at small tier to find the sweet spot.
- **Test SGCC on ping**, where the additional E↔I↔E loop creates extra positive-feedback paths.
- **Re-attempt DMTS with SGCC** — SGCC preserves cross-coupling gradient on the diagonal which the global dampener killed, so the working-memory loss should have signal.
