## Introduction

Single-MNIST training says whether the network recognises one digit; it says nothing about how its state turns over within a trial. Each test trial here is a sequence of four random MNIST digits, 50 ms each, inside a 200 ms input. Weights are frozen at single-MNIST values and the trained mem-mean readout is applied *per window*: average *v_out* over the *k*-th 50 ms slice, push through the same *W_out*, and check whether the argmax matches the digit shown in window *k*.

Chance is 10%. The ceiling is roughly the trial-level test accuracy (~88%): if the trained decoder generalises perfectly from full-trial mean to window-local mean, every window decodes as well as a static MNIST trial.

## Method

cuba / coba / ping trained on standard single-MNIST at the configured tier — recipes from [nb010](/notebooks/nb010/), [nb011](/notebooks/nb011/), [nb012](/notebooks/nb012/). Then 800 frozen-weight test trials, each four MNIST test images Poisson-encoded for 50 ms and concatenated. The mem-mean accumulator is recorded per timestep and finite-differenced to recover the window-local mean; argmax through the trained *W_out* gives a per-window prediction.

| param | value |
|---|---|
| windows × window length | 4 × 50 ms |
| total trial | 200 ms |
| trials | 800 |
| input rate | 25 Hz Poisson per pixel |
| readout | trained mem-mean *W_out*, applied per 50 ms window |

## Implications

The trial-trained mem-mean decoder is essentially a window-local decoder: averaging over 50 ms gives nearly the same readout quality as averaging over 200 ms. Each model refreshes its readable state on a sub-50-ms timescale despite very different firing-rate regimes — cuba doing it with dense activity, ping doing it with sparse rhythm. The w₀ dip is the only systematic effect and is small.

This experiment was originally designed assuming the trial-level readout would dominate, leaving the 1-of-4 ceiling (~25%) and forcing a richer probe. With a per-window readout the ceiling is gone, and the answer is that all three architectures track adequately at this 50 ms scale; the discrimination has to come from a harder task.

## Next steps

- Push window length down (10 / 25 ms) until each model breaks — the breaking point is the natural turnover scale.
- Train on the sequential task with a last-window loss to test learnable temporal binding rather than passive dynamics.
- Move to a task where the four windows must be combined (e.g. summed or compared), so the integration time matters.
