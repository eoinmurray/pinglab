## Introduction

Training on single MNIST digits tells us whether a network can recognise *a* digit. It does not tell us how quickly the network forgets one digit and locks onto the next. This entry asks that question directly: show the network four different digits back-to-back, 50 ms apiece, and ask after each 50 ms slice which digit is on screen right now.

Three networks go in: cuba, coba, and ping. All three are trained the standard way — one digit at a time — and then frozen. We do not retrain them on sequences. Instead, we reuse the readout learned from single digits and apply it window by window. A perfect readout would hit the single-digit ceiling of $\sim$88% on every window; chance is 10%.

## Method

The networks come from the standard single-MNIST recipes in [nb010](/notebooks/nb010/), [nb011](/notebooks/nb011/), and [nb012](/notebooks/nb012/). For evaluation we build 800 four-digit test trials. Each digit is Poisson-encoded for 50 ms; the four 50 ms streams are concatenated into a single 200 ms input. Weights stay frozen.

To read out window by window, the mem-mean accumulator (the running average of *v_out* the network was trained against) is logged every timestep and then finite-differenced across window boundaries. That recovers the average *v_out* during each individual 50 ms slice. The same trained *W_out* turns each slice's mean into class scores; the argmax is the prediction for that window.

| param | value |
|---|---|
| windows × window length | 4 × 50 ms |
| total trial | 200 ms |
| trials | 800 |
| input rate | 25 Hz Poisson per pixel |
| readout | trained mem-mean *W_out*, applied per 50 ms window |

Every model sits close to its single-digit baseline ($\sim$88%) on every window — the readout trained on full-trial averages works just as well on 50 ms slices. ping is a few points ahead of cuba and coba on every window. The first window (w₀) is consistently the weakest, which fits the picture of an onset transient before the recurrent state has settled into its working regime.

## Next steps

- Shrink the window (25 ms, then 10 ms) until each model breaks. The breaking point is a direct measure of the network's natural turnover speed.
- Add training on the sequential task itself, with the loss focused on the last window, to test whether the networks can *learn* temporal binding rather than just inherit it from passive dynamics.
- Move to a task that forces the windows to interact — e.g. report the sum or compare two of the digits — so integration time becomes the bottleneck.
