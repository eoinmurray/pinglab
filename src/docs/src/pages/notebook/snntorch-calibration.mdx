# snnTorch calibration

*Friday, April 17 2026*

## Introduction

The repo ships two variants of a snnTorch-style network: the in-repo *snntorch* model and *snntorch-library*, which swaps in snnTorch's own *snn.Leaky* + fast-sigmoid surrogate as a parity reference ([Eshraghian et al. 2023](https://arxiv.org/abs/2109.12894); [Neftci et al. 2019](https://arxiv.org/abs/1901.09948)). They are meant to be behaviourally equivalent at matched config — any residual gap localises to the LIF update and the surrogate-gradient details. This entry is the first head-to-head run under matched hyperparameters.

## Method

Both models were trained on a {cfg.max_samples}-sample subset of MNIST for {cfg.epochs} epochs, batch size 64, lr 0.01, using signed Kaiming-uniform feedforward init (*--kaiming-init*), dt 0.25 ms, T 600 ms, 1024 hidden units, all other flags at defaults. The only configuration difference between the two runs is the *--model* flag. Seed control in the CLI is not yet wired, so the runs draw fresh RNG state but otherwise see identical data, optimizer, and architecture.

### Runtime gap

The pinglab path took {snn.total_elapsed_s.toFixed(0)} s for {cfg.epochs} epochs; the library path took {lib.total_elapsed_s.toFixed(0)} s — a ~{(snn.total_elapsed_s / lib.total_elapsed_s).toFixed(0)}× ratio. This is far larger than the LIF-step difference alone can explain; the in-repo forward pass also records per-step membrane and firing traces that the library path skips. Part of what calibration has to disentangle is how much of the accuracy gap is genuinely the LIF step versus side-effects of recording.

## Implications

The two networks are *not* calibrated yet. Claiming that the in-repo LIF is a drop-in equivalent of snnTorch's requires matching behaviour within a small tolerance, not the {(snn.best_acc / lib.best_acc).toFixed(1)}× accuracy gap observed here. A plausible suspect list: (a) the membrane-update convention (β, decay timing, or a subtle double-integration), (b) spike-reset mechanics, (c) surrogate slope and centering, (d) *cm_back_scale* gradient-dampening interacting with the tutorial-mode path. The next pass has to localise which of these carries the bulk of the discrepancy.
