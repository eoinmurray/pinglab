## Abstract

Karpathy's *autoresearch* loop turns an AI coding agent into an overnight researcher: edit one training file, train for a fixed time budget, keep the change if a single scalar improves, revert otherwise, repeat. It works on language modelling because val-bits-per-byte has effectively unbounded headroom and clean signal. Whether it works on small biologically-structured spiking networks is the open question. This entry establishes the *baseline* the loop iterates against — a single PING/COBA network trained on the Spiking Heidelberg Digits (SHD) audio dataset with the standard leaky-integrator readout, frozen recipe (E/I strength $0.5$, ratio $2$; $128$ hidden units; $\Delta t = 2$ ms over $T = 1{,}000$ ms; Adam at $10^{-3}$; seed $42$). It is deliberately small so a full training run fits inside the autoresearch 5-minute wall-clock budget on a single GPU, and deliberately *not* state-of-the-art: the point is to give the agent enough headroom to find wins, not to compete with the literature.

## Background

SHD is the de-facto SNN classification benchmark — 20 spoken-digit classes (English + German) delivered as 700-channel cochlear event streams, $\approx 8{,}000$ training examples, $\approx 2{,}000$ test. It is genuinely event-based, which matters: the input is already temporal structure, so the architecture choices a PING agent might explore (E/I balance, time constants, synapse models, readout schemes) all have somewhere to bite. The matching MNIST recipe in this repo runs at $\Delta t = 0.1$ ms over $T = 200$ ms — $2{,}000$ timesteps per trial — and a single training run takes $\approx 1$ hour, far outside the 5-minute autoresearch budget. SHD at $\Delta t = 2$ ms over $T = 1{,}000$ ms is $500$ timesteps per trial, $\approx 4$ s per epoch on a Macbook MPS at $n_\text{hidden}=128$, which is what makes the loop feasible at all.

The autoresearch contract differs from a normal training notebook in one important way: this file *is the recipe*, and it is not edited by the agent. The agent works on its own branch (autoresearch/&lt;tag&gt;), hacks on a copy of the trainer there, and only successful experiments get promoted to a new notebook entry in this collection. Treating the baseline as immutable is what makes the comparison honest later.

## Method

1. **Dataset.** Spiking Heidelberg Digits, loaded via *load\_dataset("shd", dt\_ms=2, t\_ms=1000)*; max-samples scaled with tier (medium uses $3{,}000$ training examples, large uses $5{,}000$, extra-large uses the full $\approx 8{,}156$ train set).
2. **Architecture.** Single-hidden-layer PING/COBA network with $n_\text{hidden} = 128$, E/I split of $4{:}1$ ($96$ excitatory, $32$ inhibitory), Dale's law on. Recurrent E$\to$I and I$\to$E couplings initialised at *ei-strength* $= 0.5$ with *ei-ratio* $= 2.0$ (so $W_\text{IE} = 2\,W_\text{EI}$).
3. **Dynamics.** Conductance-based leaky integrate-and-fire neurons, integrated at $\Delta t = 2$ ms over a $T = 1{,}000$ ms window, so each trial is $500$ timesteps. Surrogate-gradient BPTT with a membrane-voltage gradient dampener of $80$.
4. **Readout.** Leaky-integrator readout (*--readout li*) — a non-spiking integrator per class with max-over-time, the standard SHD-style readout.
5. **Optimisation.** Adam at learning rate $10^{-3}$, batch size $256$, no learning-rate schedule, no early stopping. Epochs scale with tier (medium: $20$; large: $40$; extra-large: $60$). Seed $42$.
6. **Metric.** Held-out test accuracy on the SHD test set after each epoch; *best_acc* is the max across epochs. This is the single scalar the autoresearch loop will hill-climb on for any descendant experiment in this collection.
