## Abstract

The О”t audit asks whether the [nb025](/notebooks/nb025/) headline E rate is a physical (Hz) property of the trained network or an artefact of the integration timestep. The rate stays in a 9.5вЂ“14 Hz band across a 20Г— О”t sweep, accuracy holds at 88.5вЂ“90.7%, and the gamma cycle period in physical ms is invariant вЂ” but the rate dependence on О”t is non-monotonic, with О”t = 0.5 ms producing a *lower* mean rate than О”t = 0.25 ms.

## Method

**Training recipe (canonical / medium tier):**

| Parameter | Value |
|---|---|
| Integration timestep $\Delta t$ | 0.1 ms |
| Trial duration $T$ | 200 ms |
| MNIST samples (80/20 stratified split of 2000) | 1600 train / 400 test ($\approx$ 2.9% of the 70k-sample MNIST corpus) |
| Epochs | 100 |

Train one PING from scratch per $\Delta t \in \{0.05, 0.1, 0.25, 0.5, 1.0\}$ ms Г— seed $\in \{42, 43, 44\}$ = 15 cells. Total physical time $T = 200$ ms held constant (step count varies 4000 в†’ 200). Batch size 64 throughout вЂ” smaller than [nb025](/notebooks/nb025/)'s 256, but matched across the sweep so per-step compute and memory stay comparable, and the О”t = 0.05 cells (4000 timesteps Г— $N_E$ Г— $N_I$) fit in a single A100. All other PING recipe parameters held to [nb025](/notebooks/nb025/).

After training, run inference on the test set; report mean E rate (Hz), accuracy, and a single-trial raster from seed 42 per О”t for visual cycle-period inspection.
