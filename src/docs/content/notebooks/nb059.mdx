## Abstract

The first concrete test of the [PING-as-a-clock](/articles/ar062/) idea: if the architecture-generated gamma rhythm acts as a self-organised timing reference, a network that carries the loop should degrade more gracefully under per-neuron noise than the same network without it, because the inhibitory volley re-references the excitatory population every cycle. We train matched PING and COBA classifiers, then sweep membrane noise at inference and measure two things — test accuracy and the *trial-to-trial dispersion of the readout* (how much the output wobbles when the same input is presented under independent noise). The same run also logs the inhibitory spike count, which is the energy the self-clock actually costs, and turns it into a back-of-envelope ledger against a conventional clock tree. Status: **draft / spec** — the recipe below is fixed; figures populate once it is run.

## Hypothesis

Two predictions, one per half of the [ar062](/articles/ar062/) claim:

- **Reliability.** As membrane noise $\sigma$ rises, the readout dispersion $\bar{D}(\sigma)$ grows more slowly for PING than for COBA, and PING's accuracy falls more slowly. The mechanism is per-cycle resynchronisation: jitter that COBA lets accumulate over the whole trial, PING resets each gamma period.
- **Energy.** The inhibitory spike rate needed to maintain the rhythm, converted to power, is below the modelled dynamic power of a clock tree distributing the same frame rate — above some array size $N^\star$ the self-clock is the cheaper way to buy synchrony.

## Method

| Parameter | Value |
|---|---|
| Integration timestep $\Delta t$ | 0.1 ms |
| Trial duration $T$ | 200 ms |
| Dataset | MNIST (stratified 80/20 split; sample count set by tier) |
| Noise grid $\sigma$ (mV-scale, on $g_E$) | 0, 1, 2, 4, 8, 16, 32 |
| Repeats per input $K$ | 8 (independent noise draws, identical input) |
| Models | PING (loop on) and COBA (loop off), each trained at its own competent operating point |

The PING and COBA definitions and the training recipe are inherited from [nb025](/notebooks/nb025/) / [nb038](/notebooks/nb038/) — both are the same architecture, with COBA the $W^{EI}=W^{IE}=0$ limit. Training each model *with* its own loop setting removes the accuracy confound of [nb038](/notebooks/nb038/) (where switching the loop on at inference cost accuracy): here each network is competent in its own configuration, so reliability is measured as degradation relative to that network's own clean baseline.

**Noise model.** The perturbation is per-step, per-cell white noise added to the excitatory conductance $g_E$ — the simulator's *noise-std* mechanism, injected in-process at inference. Term by term, the readout under noise level $\sigma$ on trial $k$ for input $x$ is $z_k(x;\sigma)$. Note: this knob is wired into the single-trial scope path of the CLI, not into the batched evaluator, so the runner drives the forward pass in-process (the [nb038](/notebooks/nb038/) pattern) rather than via *sim --infer*. Quenched per-cell mismatch and independent Poisson jitter are available on the same forward and are listed under Next steps as further noise axes.

**Reliability metric.** For each test input $x$ and noise level $\sigma$, run $K$ independent noisy trials and collect the readout logit vectors $z_k(x;\sigma)\in\mathbb{R}^C$. Define the per-input readout dispersion

$$
D(x;\sigma) = \frac{1}{C}\sum_{c=1}^{C}\operatorname{Var}_{k}\big[z_{k,c}(x;\sigma)\big],
\qquad
\bar{D}(\sigma) = \mathbb{E}_x\!\left[\sqrt{D(x;\sigma)}\right]
$$

- $z_{k,c}(x;\sigma)$ — logit for class $c$ on noisy trial $k$ of input $x$ at noise level $\sigma$;
- $C$ — number of classes (10 for MNIST);
- $\operatorname{Var}_k$ — variance across the $K$ repeats (same input, independent noise);
- $D(x;\sigma)$ — mean across classes of that variance, i.e. the readout's wobble for one input;
- $\bar{D}(\sigma)$ — RMS readout jitter averaged over the evaluation set, the headline reliability number;
- $\mathbb{E}_x$ — average over the held-out evaluation inputs.

At $\sigma=0$ every repeat is identical, so $\bar{D}(0)=0$ for both models — a built-in sanity check. Accuracy $A(\sigma)$ is the mean over repeats of the fraction correctly classified.

**Energy ledger.** A conventional synchronous clock distributes a tick to every endpoint through a tree whose dynamic power is

$$
P_\text{clk} = \alpha\, C_\text{clk}\, V_\text{dd}^2\, f_\text{clk},
\qquad C_\text{clk} = c_\text{node}\, N
$$

- $P_\text{clk}$ — dynamic power of the clock tree;
- $\alpha = 1$ — the clock's activity factor (it toggles every cycle by construction);
- $C_\text{clk}$ — total switched capacitance, modelled as proportional to the number of synchronised endpoints $N$;
- $c_\text{node}$ — per-endpoint switched capacitance (buffer + wire + register clock pin);
- $V_\text{dd}$ — supply voltage;
- $f_\text{clk}$ — clock frequency, set equal to the gamma frame rate $f_\gamma$ for a like-for-like comparison;
- $N = N_E + N_I$ — endpoints to be kept in time.

The self-organised clock is instead paid for in inhibitory spikes:

$$
P_\text{ping} = r_I\, N_I\, E_\text{sp}
$$

- $r_I$ — measured mean inhibitory firing rate that sustains the rhythm (from this run);
- $N_I$ — number of inhibitory neurons;
- $E_\text{sp}$ — energy per spike event (synaptic operation), taken from published per-op figures for the target substrate.

Setting $P_\text{ping}=P_\text{clk}$ gives a crossover array size $N^\star$ above which the rhythm is the cheaper clock. The ledger is explicitly back-of-envelope: it uses one measured quantity ($r_I$) and two literature constants ($c_\text{node}$, $E_\text{sp}$), and is meant to locate an order of magnitude, not to model a specific chip.

## Success criteria

- $\bar{D}(0)=0$ for both models (sanity).
- PING shows a smaller dispersion slope $\mathrm{d}\bar{D}/\mathrm{d}\sigma$ and a smaller accuracy drop than COBA over the grid; if it does not, the resynchronisation story is wrong and the [ar062](/articles/ar062/) reliability claim should be retracted.
- The ledger yields a finite, plausible $N^\star$ rather than "never" — otherwise the energy claim needs a per-spike or per-node constant that does not match real hardware.

## Next steps

- Add quenched per-cell mismatch and independent Poisson jitter as further noise axes (both available on the same forward).
- Sweep $\tau_\text{GABA}$ mismatch across the I population specifically — the most clock-relevant perturbation, since it detunes the oscillators.
- If reliability holds, measure how dispersion scales with trial length $T$: the resynchronisation story predicts COBA's jitter grows with $T$ while PING's stays bounded at one cycle.
