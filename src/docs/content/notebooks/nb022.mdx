## Abstract

The single place this collection trains its canonical networks. Every cell is trained once here, to a shared artifact root, and the analysis notebooks load those weights instead of retraining their own — a train-once / reuse-many policy that replaces the older "standalone runner" rule (see [ar016](/articles/ar016/)). Status: **draft** — the registry and standard are fixed, but the cells await the canonical run: the full-MNIST reference is new, and the folded sweep cells on disk predate the gamma standard ($\tau_\text{GABA} = 6$ ms), so they will be retrained at it.

### Target trainings

Each family fixes its synaptic time constants, a timestep, a training length, and how much of MNIST it sees (of the 70,000-image pool — 60k train + 10k test, sampled then re-split 80/20). ### Cost to run

The full registry is ≈ **360 A100-GPU-hours** of work — the six canonical cells (full MNIST × 100 epochs) dominate at ≈ 186 of those hours; the sixty-one 10%-MNIST sweep cells make up the rest. A *GPU-hour* is one GPU running for one hour; the figures below scale that one number across the venues considered. Surrogate-gradient BPTT over ≈ 2000 timesteps is memory-heavy, which is why the cloud options force (or want) a large-memory card.

| Venue | Card | Price | Estimated for the run | Notes |
|---|---|---|---|---|
| Modal | A100-80GB (forced) | ≈ \$3.30 / GPU-hr | ≈ **\$1,200** | Fans out across many GPUs, so wall-clock is hours not days; spends real money per run. |
| Cambridge Wilkes3 | A100-80GB | £0.55 / GPU-hr | ≈ **£200** | Academic HPC, SLURM queue; cheapest credible option but needs account + setup. |
| Vast.ai | RTX 4090 | ≈ \$0.35 / GPU-hr | ≈ **\$60 – 130** | Cheapest cloud; the 4090 is actually fastest here (FP32 SNN), but it is one rented GPU → serial wall-clock, no Modal-style fan-out. |
| Local build | RTX 5090 | ≈ \$3,400 capital | electricity after | Continuous runs, no quota or queue; FP32 throughput above the A100; large upfront cost. |
| Owned | M5 MacBook Pro (MPS) | owned | impractical at scale | Far slower than any discrete GPU; fine for the smoke tier only. |
| Owned | 2060-class laptop | owned | impractical | OOMs on the full BPTT graph; ≈ 45 – 90 days even if it fit. |

The standing plan is **Wilkes3** for the canonical run — an order of magnitude cheaper than Modal — with Modal kept for quick one-off cells where its instant fan-out is worth the premium. Vast.ai is the fallback if the HPC account slips.

### Saved per cell

Each training writes one directory under the shared root (*src/artifacts/notebooks/training/&lt;cell&gt;/*). Every cell saves the same set of files — the contract the analysis notebooks read:

| File | What it holds |
|---|---|
| weights.pth | Trained model state — every parameter tensor of the network. |
| config.json | The full run config: model, lr, epochs, dt, t_ms, tau_gaba, w_in, ei_strength, ei_ratio, max_samples, n_params, and the git sha. |
| metrics.json | Summary plus the complete per-epoch history: best_acc, best_epoch, total_elapsed_s, and an *epochs* list (one record per epoch). |
| metrics.jsonl | The same per-epoch records, one JSON object per line — the streaming log written live during training. |
| test_predictions.json | Predicted vs true label for every image in the test split. |
| output.log | The training run's stdout. |
| run.sh | The exact train command, re-runnable. |

**metrics.json** bundles a run summary with the full per-epoch history; **metrics.jsonl** is the same per-epoch records streamed one JSON object per line (written live, so a run can be inspected before it finishes).

Top-level fields in metrics.json:

| Field | Holds |
|---|---|
| mode, model | run mode (*train*) and model name. |
| run_finished_at | UTC timestamp the run completed. |
| config | the full run config (mirror of config.json). |
| init, end | network-dynamics snapshot before / after training — each holds rate_e, rate_i, cv, act, $f_0$. |
| epochs | the list of per-epoch records (below). |
| best_acc, best_epoch | best test accuracy over the run and the epoch it occurred. |
| total_elapsed_s | total wall-clock training time. |
| perf | hardware + throughput: device, torch version, peak memory, first-epoch and warm-mean epoch time, samples/sec. |

Each per-epoch record — one entry in metrics.json's *epochs* list, one line in metrics.jsonl:

| Field | Holds |
|---|---|
| ep | epoch number. |
| acc | test accuracy (%). |
| loss, test_loss | mean train and test cross-entropy loss. |
| rate_e, rate_i | mean train-set E and I firing rate (Hz). |
| test_rate_e, test_rate_i | mean test-set E and I firing rate (Hz). |
| cv | inter-spike-interval coefficient of variation (spike-timing irregularity). |
| act | active fraction — share of hidden neurons that fired. |
| f0 | gamma peak frequency from the population-rate PSD (Hz). |
| lr | learning rate this epoch. |
| grad_norm | clipped global gradient norm, mean over the epoch's steps. |
| grad_norms | gradient norm **per weight type** — win (W_ff.0, input), wout (W_ff.1, readout), and wei (W_ei.1), wie (W_ie.1) for the trainable recurrent loop. Mirrored as gnorm__&lt;param&gt; in the jsonl. |
| grad_ratios | per-weight-type update-to-weight ratio, same keys as grad_norms. |
| weight_norms | per-weight-type Frobenius weight norm (read by the convergence audit). |
| skipped_steps | optimiser steps skipped on non-finite gradients. |
| new_best | whether this epoch set a new best accuracy. |
| samples | training samples seen this epoch. |
| elapsed_s, train_compute_s, eval_s, observe_s | cumulative wall time and the per-epoch timing breakdown. |
