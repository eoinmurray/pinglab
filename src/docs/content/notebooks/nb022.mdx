## Abstract

The one place this collection trains its networks: every cell is trained once, to a shared root, and the analysis notebooks load those weights instead of retraining (train-once / reuse-many, see [ar016](/articles/ar016/)). Status **draft** — nothing is trained to the gamma standard yet; the cells await the canonical run.

## Methods

### 1. The compute cost

Each trial is surrogate-gradient backprop-through-time: 2000 timesteps ($T = 200$ ms at $\Delta t = 0.1$ ms) over a 1280-neuron conductance network, with the loss flowing back through all of it. What makes it costly:

- **Fine timestep, forced by biophysics.** Conductance-based (COBA) synapses inject $g\,(V - E)$, which turns the membrane equation stiff — so $\Delta t = 0.1$ ms, 10× finer than a current-based model. Not optional: the gamma rhythm lives in those terms.
- **Sequential in time.** The 2000 steps are a dependency chain no hardware parallelises, so the job is **memory-bandwidth-bound** — a GPU beats a CPU only ≈ 10×, not the 50–100× of large matmuls.
- **Memory-heavy.** The backward pass stores every step's activations — ≈ 12 GB at batch 256.
- **Scale.** 67 cells × 50 epochs ≈ **128 A100-GPU-hours** (≈ 33.9M sample-forwards) — ≈ 110 days on CPU, so every cell trains on a GPU.

### 2. Gold star trainings

67 cells across five families. Each fixes its time constants, timestep, length, and how much of the 70,000-image MNIST pool it sees (sampled, re-split 80/20). The standard is $\tau_\text{AMPA} = 2$ ms and $\tau_\text{GABA} = 6$ ms (loop in gamma, ≈ 44 Hz); the τ_GABA ladder varies the inhibitory decay, [nb044](/notebooks/nb044/)'s audit the timestep.

| Family | $\tau_\text{GABA}$ (ms) | $\Delta t$ (ms) | Epochs | MNIST | Cells | At spec |
|---|---|---|---|---|---|---|
| canonical (θ_u off) | 6 | 0.1 | 50 | all (70k) | 6 | 0 / 6 |
| θ_u sweep | 6 | 0.1 | 50 | 10% | 16 | 0 / 16 |
| τ_GABA ladder | 4.5 – 27 | 0.1 | 50 | 10% | 18 | 0 / 18 |
| Δt sweep | 6 | 0.05 – 1.0 | 50 | 10% | 15 | 0 / 15 |
| init variants | 6 | 0.1 | 50 | 10% | 12 | 0 / 12 |
| **total** | | | | | **67** | **0 / 67** |

The **canonical** family (coba and ping at θ_u = off) is the full-MNIST reference the 10%-trained sweeps are read against. *At spec* is trained to the gamma standard above — **none yet**. 61 cells sit on disk from the original notebooks, but at the old standard (2.9–5% MNIST, $\tau_\text{GABA} = 9$ ms), so the canonical run trains all 67 fresh.

Choices behind the table:

- **50 epochs — halved from 100.** Test accuracy plateaus by ≈ 15–20 epochs ([nb024](/notebooks/nb024/)), so 50 still leaves a ≈ 30-epoch post-convergence tail — enough to resolve the *dynamics* the collection studies (firing-rate drift, confidence inflation), which the earlier 100-epoch budget captured with room to spare. Halving the epoch count scales the run almost linearly, from ≈ 255 to ≈ 128 GPU-hours.
- **3 seeds — necessary.** Baselines train three seeds so cross-seed spread is measurable and the headline effects (e.g. PING's rate attractor holding across seeds, not one-run luck) are shown to be robust; the variance band is load-bearing in the analysis. Sweep cells are single-seed, where the *trend* across the swept value carries the claim instead.
- **Full MNIST vs 10%.** The canonical reference carries the headline accuracy and rate numbers, so it sees all 70k images; the sweeps only need the trend across their parameter, so 10% is enough and cuts their cost roughly tenfold.

### 3. Compute options

Bandwidth is the main driver, but not the whole story: the RTX 4090 below has half the A100's bandwidth yet measured *faster* here, so Ada's large L2 and raw throughput also count at this batch and model size. Costs and wall-clocks are for the full 50-epoch registry (≈ 33.9M sample-forwards). Measured where the card was to hand (A100, 4090, benjy, Mac), projected otherwise (5090):

| Option | GPU (bandwidth) | Samples/s | Full-run cost | Wall-clock |
|---|---|---|---|---|
| Modal | A100 (2.0 TB/s) | 74 | ≈ **\$425** | ≈ half a day (fans out) |
| RunPod / Vast.ai | RTX 4090 fleet (1.0 TB/s) | 100 (meas.) | ≈ **\$30 – 65** | ≈ 4 d serial / hours fanned out |
| Cambridge Wilkes3 | A100 | 74 | ≈ **£70** | **offline through 2026** |
| benjy (CUED, shared) | A6000 (768 GB/s) | 26.5 | **£0** | ≈ 15 days |
| hacktar (CUED, shared) | 2080 Ti / 1080 Ti (11 GB) | 6.8 | **£0** | ≈ 58 d — **unusable** |
| Owned workstation | RTX 5090 (1.8 TB/s) | ≈ 150 (proj.) | ≈ **£4,700** once | ≈ 2.5 days |
| Mac (small runs) | Apple MPS | 36 | — | ≈ 11 days |

The measured 4090 (≈ 100 samples/s, RunPod) is the anchor: it beat its own bandwidth projection (≈ 37) and the A100, so the 5090 estimate is scaled from it (≈ 1.5×, deliberately conservative) rather than from raw bandwidth. Wilkes3 would be cheapest-and-fast, but the West Cambridge data centre is mid power-and-cooling-upgrade with recurring outages through 2026. Of the CUED boxes, **hacktar** (11 GB cards) is unusable — the graph needs 12 GB and its 1080 Ti is too old for torch.compile — while **benjy** works but is a shared single GPU (a fortnight-long lock is not on) whose mid-2024 driver forces an older PyTorch build. So the one-off canonical run is a cloud fan-out (a RunPod / Vast.ai 4090 fleet at ≈ \$30 – 65, cheaper than Modal), small runs sit on benjy or the Mac, and repeat runs point to a dedicated **5090** — no queue, cost, or driver friction.

### 4. Saved per cell

Each training writes one directory under the shared root (*src/artifacts/notebooks/training/&lt;cell&gt;/*), with the same files every time — the contract the analysis notebooks read:

| File | What it holds |
|---|---|
| weights.pth | Trained model state — every parameter tensor of the network. |
| config.json | The full run config: model, lr, epochs, dt, t_ms, tau_gaba, w_in, ei_strength, ei_ratio, max_samples, n_params, and the git sha. |
| metrics.json | Summary plus the complete per-epoch history: best_acc, best_epoch, total_elapsed_s, and an *epochs* list (one record per epoch). |
| metrics.jsonl | The same per-epoch records, one JSON object per line — the streaming log written live during training. |
| test_predictions.json | Predicted vs true label for every image in the test split. |
| output.log | The training run's stdout. |
| run.sh | The exact train command, re-runnable. |

**metrics.json** bundles a run summary with the full per-epoch history; **metrics.jsonl** streams those same records live, one per line, so a run can be read before it finishes.

Top-level fields in metrics.json:

| Field | Holds |
|---|---|
| mode, model | run mode (*train*) and model name. |
| run_finished_at | UTC timestamp the run completed. |
| config | the full run config (mirror of config.json). |
| init, end | network-dynamics snapshot before / after training — each holds rate_e, rate_i, cv, act, $f_0$. |
| epochs | the list of per-epoch records (below). |
| best_acc, best_epoch | best test accuracy over the run and the epoch it occurred. |
| total_elapsed_s | total wall-clock training time. |
| perf | hardware + throughput: device, torch version, peak memory, first-epoch and warm-mean epoch time, samples/sec. |

Each per-epoch record — one entry in metrics.json's *epochs* list, one line in metrics.jsonl:

| Field | Holds |
|---|---|
| ep | epoch number. |
| acc | test accuracy (%). |
| loss, test_loss | mean train and test cross-entropy loss. |
| rate_e, rate_i | mean train-set E and I firing rate (Hz). |
| test_rate_e, test_rate_i | mean test-set E and I firing rate (Hz). |
| cv | inter-spike-interval coefficient of variation (spike-timing irregularity). |
| act | active fraction — share of hidden neurons that fired. |
| f0 | gamma peak frequency from the population-rate PSD (Hz). |
| lr | learning rate this epoch. |
| grad_norm | clipped global gradient norm, mean over the epoch's steps. |
| grad_norms | gradient norm **per weight type** — win (W_ff.0, input), wout (W_ff.1, readout), and wei (W_ei.1), wie (W_ie.1) for the trainable recurrent loop. Mirrored as gnorm__&lt;param&gt; in the jsonl. |
| grad_ratios | per-weight-type update-to-weight ratio, same keys as grad_norms. |
| weight_norms | per-weight-type Frobenius weight norm (read by the convergence audit). |
| skipped_steps | optimiser steps skipped on non-finite gradients. |
| new_best | whether this epoch set a new best accuracy. |
| samples | training samples seen this epoch. |
| elapsed_s, train_compute_s, eval_s, observe_s | cumulative wall time and the per-epoch timing breakdown. |
