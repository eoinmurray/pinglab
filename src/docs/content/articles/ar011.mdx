Every figure on this site is generated by one command-line tool. The notebooks under [/notebooks](/notebooks/) are thin wrappers — each runner just hardcodes a recipe and shells out — and the article-level demos sit on the same primitives. This entry is the reference for that tool: every subcommand, every flag, every default. It is not a tutorial; it is the dictionary you reach for when a notebook mentions *--ei-strength 0.5* and you want to know exactly what that did.

## Invocation

The entry point lives at *src/cli/cli.py* and is invoked as a script:

```
uv run python src/cli/cli.py <subcommand> [flags…]
```

There are two subcommands — *sim* and *train* — plus the bare invocation *python src/cli/cli.py*, which prints the top-level help. Every subcommand also responds to *--help* with its own complete flag listing. There is no global config file: the source of truth for every parameter is the command line. Reproducibility is delegated to two artifacts the CLI writes into the output directory on every run — *config.json* (a serialisation of the parsed argparse namespace plus provenance metadata) and *run.sh* (the literal shell command that was invoked). Together they let any downstream subcommand re-derive the exact state via *--from-dir*.

## The two subcommands

### *sim*

Run one forward pass. By default it prints firing-rate metrics only — no plots, no video, no weights — the cheapest mode, used by the test suite and the dt-stability checks in [ar003](/articles/ar003/). Three flags add outputs; each does its own forward pass and reports metrics:

- *--image* — also save a still oscilloscope figure (E and I rasters, weight histograms, PSD panel). The input mode is whatever *--input* / *--dataset* dictate; the default render lands at *src/artifacts/oscilloscope/* unless *--out-dir* is set.
- *--video* — sweep one parameter linearly between *--scan-min* and *--scan-max* over *--frames* values and stitch the result into an MP4. The scan variable is *--scan-var* (legal set registered in *src/cli/scan.py*; see *Scan variables* below). This is how the gamma-gated-sparsity collection presents its parameter sweeps — the eye picks up a bifurcation in a sweep far faster than in a row of static panels. With *--input dataset* it also emits an *init_d0s0.png* snapshot alongside the video.
- *--infer* — load trained weights (*--from-dir* / *--load-weights*) and evaluate test-set accuracy, writing *results.json*; pair with *--image* for a snapshot of the trained network.

*sim* flags:

| flag | default | description |
|---|---|---|
| *--image* | off | Render the still snapshot (above). |
| *--video* | off | Render the parameter sweep (above). |
| *--infer* | off | Evaluate test-set accuracy with trained weights (above). |
| *--max-samples INT* | all | [*--infer*] Cap the evaluation set to N samples. |
| *--from-dir DIR* | — | Inherit *config.json* + auto-detect *weights.pth* from a training run directory. Any CLI flag explicitly passed overrides the inherited value. |
| *--load-weights PATH* | — | Direct path to a *weights.pth* file (*--from-dir* sets the auto-detected path, which *--load-weights* then overrides). |
| *--scan-var NAME* | *stim-overdrive* | [*--video*] Parameter to sweep. Full list below. |
| *--scan-min FLOAT* | 1.0 | [*--video*] Sweep start value, in the variable's natural units. For *digit*, an integer class. |
| *--scan-max FLOAT* | 50.0 | [*--video*] Sweep end value. For *digit*, an integer class. |
| *--frames INT* | 10 | [*--video*] Number of frames to render. For *digit*, overridden by the scan range. |
| *--frame-rate INT* | 10 | [*--video*] Output MP4 fps. |

### *train*

Surrogate-gradient BPTT training loop. Writes *weights.pth*, *metrics.json*, a per-step *metrics.jsonl*, *test\_predictions.json*, plus an optional per-epoch video or image series. *--epochs 0* runs the init snapshot only — useful as a probe.

*train*-only flags:

| flag | default | description |
|---|---|---|
| *--lr FLOAT* | 0.01 | Adam learning rate. PING typically needs 0.0001. |
| *--epochs INT* | 0 | Number of epochs. 0 = init-snapshot probe only. |
| *--batch-size INT* | 64 | DataLoader batch size. |
| *--max-samples INT* | all | Cap dataset to N samples for smoke tests. |
| *--observe \{video, images\}* | off | Per-epoch oscilloscope artifact. *video* emits one MP4 with one frame per epoch; *images* emits one PNG per epoch. |
| *--frame-rate INT* | 10 | fps for *--observe video*. |
| *--v-grad-dampen FLOAT* | 80.0 | Dampening factor on the COBA membrane gradient. Theory in [ar015](/articles/ar015/). |
| *--fr-reg-upper-theta FLOAT* | 0 (off) | Upper-bound target spike count per neuron per trial (θ\_u). Adds *s\_u · Σ relu(⟨z\_i⟩ − θ\_u)²* to the loss. Cramer et al. SHD RSNN: 100. |
| *--fr-reg-upper-strength FLOAT* | 0 | Coefficient s\_u on the upper regulariser. Cramer et al.: 0.06. |
| *--fr-reg-mode \{per-neuron, population\}* | *per-neuron* | Pooling axis for the regulariser. *per-neuron* concentrates pressure on the highest-firing cells (the Cramer recipe); *population* uses a single scalar over the grand mean across batch and neurons, distributing pressure uniformly. |
| *--tau-gaba FLOAT* | 9.0 ms | Override the GABA synaptic decay τ\_GABA. Default = *models.py*'s *tau\_gaba* (Börgers / Buzsaki-Wang range). nb041 sweeps this across \{4.5, …, 27\} ms while training PING from scratch; the realised gamma frequency f\_γ tracks 1/τ\_GABA. |

## Shared flag groups

The following flag groups are attached to every subcommand via a shared argparse parent.

### Network

| flag | default | description |
|---|---|---|
| *--model \{ping\}* | *ping* | Architecture. *ping* is the COBANet with E↔I coupling (the canonical gamma-generating substrate); with *--ei-strength 0* the inhibitory loop is silenced for a no-rhythm control. |
| *--n-hidden INT [INT …]* | dataset-dependent | Hidden layer sizes. One integer = single layer; multiple stacks layers. Defaults: scikit 64, mnist 1024, smnist 32, shd 256. |
| *--readout \{rate, li, spike-count, mem-mean\}* | *rate* | Output stage. *rate* sums last-hidden spikes and projects linearly at the final timestep. *li* is a non-spiking leaky integrator per class with max-over-time — the SHD-standard readout. *spike-count* and *mem-mean* are alternative pooling rules used in dt-stability sweeps. |
| *--dales-law* / *--no-dales-law* | on | Enforce Dale's law (non-negative weights) or allow signed weights. *--no-dales-law* is used for balanced-network experiments where weights may take either sign. |
| *--ei-layers INT [INT …]* | all layers | 1-indexed list of hidden layers that get E-I structure. PING only. |
| *--ei-strength FLOAT* | 0.5 | E-I coupling strength s. Sets W\_EI = s and W\_IE = s·ratio. |
| *--ei-ratio FLOAT* | 2.0 | W\_IE / W\_EI. |
| *--ei-sparsity FLOAT* | 0.0 | Sparsity of the recurrent E↔I matrices (W\_EE / W\_EI / W\_IE / W\_II): fraction of entries zeroed, survivors rescaled by 1/(1−s) to preserve expected drive. Use ≈ 1 − K/N for Brunel/Vreeswijk sparse random connectivity. |
| *--exact-k* | off | Fixed-fan-in (exact-K) recurrent connectivity instead of per-entry Bernoulli: every post cell draws exactly K = round((1−ei\_sparsity)·N\_pre) inputs, removing cell-to-cell fan-in variance. No effect unless *--ei-sparsity* > 0. |
| *--w-in-sparsity FLOAT* | 0.95 | Fraction of input weights zeroed at init — the network learns from a sparse subset of input units. |
| *--dt FLOAT* | 0.25 | Integration timestep (ms). |
| *--t-ms FLOAT* | 200 | Total trial duration (ms). For synthetic-step inputs, must exceed *STEP\_ON\_MS* (200 ms) or the stimulus window never opens; the CLI warns in this case. |
| *--tau-mem FLOAT* | 10 ms | Membrane time constant τ\_mem (*models.py*'s *tau\_snn*). Cramer et al. SHD: 20 ms. |
| *--tau-syn FLOAT* | 2 ms | Synaptic time constant τ\_syn for the exponential synapse. Cramer et al. SHD: 10 ms. Only affects models with *exponential\_synapse=True* (COBA / PING). |
| *--readout-tau-out FLOAT* | 2 ms | Output-LIF τ\_out for the *spike-count* readout (default = *models.py*'s *tau\_out\_ms*). Smaller values prevent saturation under high-rate hidden drive at coarse dt. No-op for *rate* / *li*. |
| *--readout-w-out-scale FLOAT* | 1.0 | Multiplicative scalar applied to the readout matrix (and bias if present) after *build\_net*. Use to compensate for low hidden firing rate under *mem-mean* / *spike-count* readouts. Train-mode only. |
| *--surrogate-slope FLOAT* | 5.0 | Fast-sigmoid surrogate-gradient slope β (default = *models.py*'s *SURROGATE\_SLOPE*). Cramer et al. use 40 for SHD RSNNs. |
| *--device \{cpu, mps, cuda\}* | auto | Compute device. Auto-detected if unset: cuda &gt; mps &gt; cpu. |

### Input

| flag | default | description |
|---|---|---|
| *--input \{synthetic-conductance, synthetic-spikes, dataset\}* | *synthetic-spikes* | Stimulus regime. *synthetic-spikes* is Poisson at *--input-rate* Hz. *synthetic-conductance* is a step current with *--stim-overdrive* multiplier. *dataset* draws from *--dataset*. **Auto-flip:** if *--dataset*, *--digit*, or *--sample* are passed explicitly and *--input* is left at the default, the parser silently flips *--input* to *dataset* — closing the footgun where *image --dataset mnist --digit 3* used to run the synthetic-spikes branch and ignore the digit. |
| *--input-rate FLOAT* | 25 | Baseline input rate (Hz). |
| *--stim-overdrive FLOAT* | 1.0 | Stimulus multiplier. |
| *--digit INT* | 0 | Dataset class (0–9). |
| *--sample INT* | 0 | Sample index within the class. |
| *--dataset \{scikit, mnist, smnist, shd\}* | *scikit* | *scikit* is the 8×8 sklearn digits; *mnist* is full 28×28; *smnist* serialises MNIST row-by-row over time; *shd* is the Spiking Heidelberg Digits audio dataset (700 input channels). SHD is cached under *\$PINGLAB\_SHD\_DIR* or a default site location. |
| *--independent-drive RATE G\_PER\_SPIKE* | off | Per-E-cell independent Poisson drive (bypasses W\_in): N\_E uncorrelated streams at RATE Hz, each spike adding G\_PER\_SPIKE μS of g\_E. synthetic-spikes mode. For Brunel/Vreeswijk experiments with zero cross-cell input correlation. |
| *--independent-drive-i RATE G\_PER\_SPIKE* | off | As *--independent-drive* but targets the I population directly, so I-cell fluctuations are no longer driven solely by E spikes via W^EI. Needed for the full V&S asynchronous-irregular state. |
| *--lyapunov-eps FLOAT* | 0 (off) | If > 0 (synthetic-spikes *--image* mode), rerun the forward pass with all membranes ε-perturbed at t=0 and save the divergence curve ‖ΔV(t)‖ to *snapshot.npz*. Its growth rate is the max Lyapunov exponent: positive for the chaotic V&S state, ≈ 0 for cycle-locked PING. |

### Weights (advanced)

| flag | default | description |
|---|---|---|
| *--w-in MEAN [STD]* | *0.3 0.06* | Input fan-in init. Single value sets STD = MEAN × 0.1. |
| *--w-ei MEAN STD* | from *--ei-strength* | Override the W\_EI init. |
| *--w-ie MEAN STD* | from *--ei-strength* / *--ei-ratio* | Override the W\_IE init. |
| *--w-ii MEAN STD* | *0 0* | W\_II (I→I) init. Off by default (canonical PING has no I→I). Enable for Brunel/Vreeswijk balanced-network experiments. |
| *--trainable-w-ei* | frozen | Promote E→I from frozen to gradient-carrying. Asks whether the optimiser will discover the PING-loop weights from scratch. |
| *--trainable-w-ie* | frozen | Promote I→E. Same question, opposite direction. The nb049 result *gradient descent dismantles PING* comes from flipping *--trainable-w-ei* and *--trainable-w-ie* on simultaneously. |

## Scan variables

The legal values of *--scan-var* (used with *sim --video*) are registered in *SCAN\_DEFAULTS* in *src/cli/scan.py*. Each entry pairs a default centre value with a unit label used for axis annotation.

| variable | default | units / notes |
|---|---|---|
| *stim-overdrive* | 1.0 × | input multiplier |
| *tau\_gaba* | 9.0 ms | GABA synaptic decay |
| *tau\_ampa* | 2.0 ms | AMPA synaptic decay |
| *w\_ei\_mean* | *W\_EI[0]* | W\_EI mean (μS) |
| *w\_ie\_mean* | *W\_IE[0]* | W\_IE mean (μS) |
| *ei\_strength* | — | joint scaling of W\_EI and W\_IE (dimensionless) |
| *spike\_rate* | — | input Poisson rate (Hz) |
| *bias* | — | DC bias conductance (μS) |
| *dt* | — | integration timestep (ms) |
| *digit* | — | dataset class (integer) |
| *noise* | — | additive Poisson noise on input (Hz) |

The *w\_ei\_mean* and *w\_ie\_mean* sweeps scale the existing matrix rather than re-sampling it, so within-sweep variance stays comparable. All other sweeps re-derive the parameter and call into *patch\_dt* if the value affects the integration step.

## Inheritance via *--from-dir*

The trick that makes the notebook chain work is *--from-dir*. Every *train* run writes a *config.json* alongside its *weights.pth*; *sim* (with *--image* / *--video* / *--infer*) can read that directory and inherit the model, hidden sizes, dataset, E-I parameters, input rate, and seed from it. Any flag the user passes explicitly on the command line wins over the inherited value — the parser builds the set of CLI-explicit flags from *sys.argv* before applying inheritance.

The inheritance map is defined in *\_apply\_from\_dir* and covers:

*model, dt, t\_ms, dataset, hidden\_sizes, ei\_strength, ei\_ratio, sparsity, w\_in\_sparsity, w\_in, input\_rate, max\_samples, dales\_law, ei\_layers, seed.*

*--out-dir* is also auto-set to a sibling *infer/* subdirectory of the source run when *sim --infer* inherits via *--from-dir*, unless overridden.

Legacy model names in older *config.json* files are remapped via *LEGACY\_MODEL\_ALIASES* with a one-line stderr note. Old configs missing the *dales\_law* field trigger a warning prompting the user to pass it explicitly or retrain.

## Artifacts and provenance

Every subcommand calls *save\_run\_artifacts* on entry, which produces three files in *--out-dir*:

- *config.json* — The parsed argparse namespace, augmented with a provenance block from *runlog.provenance()* containing *git\_sha*, *torch\_version*, *device*, *python\_env\_hash*, *run\_id*, and *started\_at*. Used by *--from-dir* and by the notebook landing-page metadata extractors.
- *run.sh* — The literal *sys.argv* joined with spaces and prefixed with *\#!/bin/bash*. Re-running this script reproduces the run bit-for-bit (modulo seeded RNG state, which is also in *config.json*).
- *output.log* — File handler attached to the *oscilloscope* logger. ANSI escape sequences are stripped from the file but preserved on stdout, so terminals see colour while the log stays grep-friendly.
