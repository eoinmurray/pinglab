### 1. Damping requirements per architecture (PING / COBA) and d=1 PING retrain

**Manuscript sites:** §5.4 "Gradient damping for the PING configuration" paragraph (asserted clause "The COBA configuration is trainable at the module default $d = 80$; the PING configuration is not"); §2.4 paragraph 3, §3 para 4, §3 limitations paragraph (the §2.4 released-loop result is currently disclosed as conditional on the $d = 1000$ damping scheme).

**Current disclosure:** the paper asserts a damping requirement asymmetry between PING ($d = 1000$) and COBA ($d = 80$ default would work) without illustrating it, and concedes the §2.4 loop-pruning result might be driven by 1/$d^{2000}$ gradient attenuation rather than by gradient descent genuinely preferring loopless networks.

**Notebook work:** new standalone notebook running both characterisation and the §2.4 d=1 retrain together — the two questions share the same damping-sweep scaffold and are best done in one place.

Training conditions (each at three seeds):
- PING, $d = 80$ — tests whether the paper's "PING is unstable at COBA default" assertion holds; if PING trains, $d = 1000$ is over-engineering and the methods simplify.
- PING, $d = 1000$ — current canonical, baseline reference.
- COBA, $d = 1$ — tests whether COBA needs any damping at all; if it trains, the §2.4 d=1 retrain becomes viable.
- PING released-loop ($W_{ei}, W_{ie}$ trainable), $d = 1$ (or the lowest $d$ that lets COBA train) — repeats the §2.4 nb049 second experiment without the 1/$d^{2000}$ confound.

Outputs: per-epoch test accuracy, gradient norm, training stability indicator (whether `nan`/`inf` appears in any layer's gradient), and for the released-loop condition the rhythmicity score $R$ trajectory.

Result interpretation:
- If both PING fixed-loop conditions ($d = 80$ vs $d = 1000$) train and reach similar accuracy: drop the $d = 1000$ choice in favour of $d = 80$ in §5.4, simplifying the methods.
- If PING at $d = 80$ fails (paper's current claim): §5.4 has its evidence, and the released-loop retrain still proceeds at whatever $d$ keeps COBA stable.
- If the released-loop $R$ trajectory still collapses to ≈0.04–0.18 at low $d$: the §2.4 conditionality clause can be deleted from the manuscript.
- If the released loop survives at low $d$: the §2.4 framing needs weakening — gradient descent could not previously *find* PING because the damping scheme was prohibitively strong.

**Where:** new notebook (suggested slug: nb050). Requires [nb022](/notebooks/nb022/) extensions to register the new training conditions.
