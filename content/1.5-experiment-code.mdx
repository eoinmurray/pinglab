Each experiment follows a consistent pattern: a YAML config, a Pydantic model,
a simulation loop, and sub-experiments that sweep parameters and save figures.

## Directory Structure

```
experiments/<name>/
├── config.yaml      # All parameters
├── experiment.py    # Entry point
└── lib/
    ├── model.py     # Pydantic config schema
    ├── hotloop.py   # Core simulation call
    └── experiment_N.py  # Sub-experiments
```

## The Config File

Everything lives in `config.yaml`. The `base` section defines network parameters,
while `experiment_N` sections define per-experiment sweeps.

```yaml
base:
  dt: 0.1           # Time step (ms)
  T: 1000.0         # Simulation duration (ms)
  N_E: 800          # Excitatory neurons
  N_I: 200          # Inhibitory neurons
  g_ei: 1.5         # E→I synaptic weight
  g_ie: 1.8         # I→E synaptic weight
  # ... LIF parameters, delays, heterogeneity

default_inputs:
  I_E: 1.7          # Input to E neurons
  I_I: 0.7          # Input to I neurons
  noise: 0.8        # Noise amplitude

experiment_1:
  I_E: 1.2          # Fixed for this sweep
  linspace:         # Sweep g_ei
    start: 1.2
    stop: 1.5
    num: 3

plotting:
  raster:
    start_time: 500.0
    stop_time: 1000.0
```

## The Pydantic Model

`lib/model.py` defines a schema that validates the YAML and provides typed access.

```python
from pinglab.types import ExperimentConfig, LinspaceConfig
from pydantic import BaseModel

class LocalExperimentConfig(BaseModel):
    I_E: float | None = None
    g_ei: float | None = None
    linspace: LinspaceConfig | None = None

class LocalConfig(ExperimentConfig):
    experiment_1: LocalExperimentConfig
    experiment_2: LocalExperimentConfig
    # ...
```

`ExperimentConfig` from pinglab provides the `base`, `default_inputs`, and
`plotting` sections. Each experiment extends it with experiment-specific fields.

## The Simulation Loop

`lib/hotloop.py` wraps the pinglab library to run a single simulation.

```python
from pinglab import run_network
from pinglab.inputs import tonic
from pinglab.types import NetworkResult

def hotloop(cfg):
    config = cfg["config"]
    g_ei = cfg["g_ei"]
    I_E = cfg["I_E"]

    external_input = tonic(
        N_E=config.base.N_E,
        N_I=config.base.N_I,
        I_E=I_E,
        I_I=config.default_inputs.I_I,
        noise_std=config.default_inputs.noise,
        num_steps=int(config.base.T / config.base.dt),
        seed=config.base.seed,
    )

    run_cfg = config.base.model_copy(update={"I_E": I_E, "g_ei": g_ei})
    result: NetworkResult = run_network(run_cfg, external_input=external_input)

    return result
```

The function takes a dict with config and sweep parameters, returns a
`NetworkResult` containing spikes and traces.

## Sub-Experiments

Each `lib/experiment_N.py` sweeps a parameter, runs simulations in parallel,
and saves figures.

```python
from pathlib import Path
import numpy as np

from pinglab.plots import save_raster
from pinglab.utils import slice_spikes
from pinglab.multiprocessing import parallel
from .hotloop import hotloop
from .model import LocalConfig

def experiment_1(config: LocalConfig, data_path: Path) -> None:
    # Build parameter grid
    cfgs = [
        {"config": config, "I_E": config.experiment_1.I_E, "g_ei": value}
        for value in np.linspace(
            config.experiment_1.linspace.start,
            config.experiment_1.linspace.stop,
            config.experiment_1.linspace.num,
        )
    ]

    # Run in parallel
    results = parallel(hotloop, cfgs, label="Experiment 1")

    # Save outputs
    for i, result in enumerate(results):
        sliced = slice_spikes(
            result.spikes,
            start_time=config.plotting.raster.start_time,
            stop_time=config.plotting.raster.stop_time,
        )
        save_raster(
            sliced,
            path=data_path / f"experiment_1_raster_g_ei_{i + 1}.png",
            label=f"g_ei={cfgs[i]['g_ei']:.2f}",
        )
```

## The Entry Point

`experiment.py` loads config, clears old artifacts, and runs all sub-experiments.

```python
import sys
from pathlib import Path
import shutil
import yaml

sys.path.insert(0, str(Path(__file__).parent.parent))
from settings import ARTIFACTS_ROOT

from lib.model import LocalConfig
from lib.experiment_1 import experiment_1
from lib.experiment_2 import experiment_2
# ...

def main():
    # Output to content/_artifacts/<experiment_name>/
    data_path = ARTIFACTS_ROOT / Path(__file__).parent.name
    if data_path.exists():
        shutil.rmtree(data_path)
    data_path.mkdir(parents=True, exist_ok=True)

    # Load and validate config
    config_path = Path(__file__).parent / "config.yaml"
    with config_path.open() as f:
        data = yaml.safe_load(f)
    config = LocalConfig.model_validate(data)

    # Run all sub-experiments
    experiment_1(config, data_path)
    experiment_2(config, data_path)
    # ...

if __name__ == "__main__":
    main()
```

## Running an Experiment

```bash
python experiments/sweeps/experiment.py
```

Outputs appear in `content/_artifacts/sweeps/` and are referenced by the
documentation via `<Gallery>` components.

## Adding a New Experiment

1. Create `experiments/<name>/` with `config.yaml`, `experiment.py`, and `lib/`
2. Define your config schema in `lib/model.py` extending `ExperimentConfig`
3. Write `lib/hotloop.py` if you need custom simulation logic
4. Add `lib/experiment_N.py` files for each parameter sweep
5. Wire them together in `experiment.py`
6. Reference outputs in `content/*.mdx` via artifact paths
