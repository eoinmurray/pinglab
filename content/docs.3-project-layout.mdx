Pinglab is organized into three interconnected parts: a Python library for neural
simulation, experiment scripts that generate results, and documentation that
presents them.

## Directory Structure

```
pinglab/
├── pinglab/          # Python library (workspace member)
├── experiments/      # Reproducible research scripts
├── content/          # Documentation site (you are here)
├── tools/            # Helper scripts and utilities
└── dist/             # Built static site output
```

## The Library (`pinglab/`)

A reusable Python package for PING circuit simulation and analysis.

```
pinglab/src/pinglab/
├── lib/              # LIF neuron and synapse models
├── run/              # Network simulation engine
├── inputs/           # External stimulus generation
├── analysis/         # Spike metrics (rate, CV, PSD, synchrony)
├── plots/            # Visualization utilities
└── types.py          # Pydantic data structures
```

Key exports: `run_network`, `types`, `analysis`, `plots`, `utils`

## Experiments (`experiments/`)

Python scripts that use the pinglab library to run simulations and generate
figures. Each experiment has:

- `config.yaml` — network parameters and sweep ranges
- `experiment.py` — entry point
- `lib/` — experiment-specific code

Current experiments:

| Directory           | Purpose                          | Artifacts          |
|---------------------|----------------------------------|--------------------|
| `metrics/`          | Population metrics analysis      | `metrics/`         |
| `sweeps/`           | Parameter sweeps (g_ei, I_E, noise) | `sweeps/`        |
| `phase-dependence/` | Phase-dependent gating           | `phase-dependence/`|
| `single-image/`     | Single-image readout             | `single-image/`    |
| `coupled-networks/` | Precision-gated communication    | `coupled-networks/`|

## Content (`content/`)

Veslx-based documentation. MDX files reference experiment outputs via the
`_artifacts/` directory.

```
content/
├── veslx.yaml        # Site configuration
├── docs.*.mdx        # Background and meta content
├── exp.*.mdx         # Experiment results and analysis
└── _artifacts/       # Generated figures from experiments
    ├── metrics/          # Metrics artifacts
    ├── sweeps/           # Sweeps artifacts
    ├── phase-dependence/ # Phase dependence artifacts
    ├── single-image/     # Single-image artifacts
    └── coupled-networks/ # Coupled-networks artifacts
```

## Data Flow

```
pinglab library
      ↓
experiments (run simulations)
      ↓
content/_artifacts (save figures)
      ↓
content/docs.*.mdx and content/exp.*.mdx (display and explain)
      ↓
veslx build → dist/ (static site)
```

Experiments write directly to `content/_artifacts/` via a shared setting:

```python
# experiments/settings.py
ARTIFACTS_ROOT = ROOT / "content" / "_artifacts"
```

This keeps generated figures version-controlled alongside the documentation that
references them.

## Running Things

**Build the docs:**
```bash
npm run dev    # development server
npm run build  # production build → dist/
```

**Run an experiment:**
```bash
python experiments/sweeps/experiment.py
```

Experiment outputs appear in `content/_artifacts/` and are picked up by the
documentation on next build.
