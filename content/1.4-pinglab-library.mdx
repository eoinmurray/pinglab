<FrontMatter/>

Pinglab is a conductance-based neural network simulation library for modeling leaky integrate-and-fire (LIF) networks with excitatory (E) and inhibitory (I) populations. It provides tools for simulating spiking neural networks with realistic biophysical properties, along with comprehensive analysis and visualization capabilities.

## Installation

```bash
pip install pinglab
```

**Requirements:** Python ≥ 3.10

**Dependencies:**
- NumPy (≥2.2.6)
- Matplotlib (≥3.10.0)
- Pydantic (≥2.12.4)
- PyYAML (≥6.0.2)
- tqdm (≥4.67.1)

## Core Data Structures

### Spikes

Container for spike event data:

```python
from pinglab.types import Spikes

spikes = Spikes(
    times=np.array([...]),      # Spike times in ms
    ids=np.array([...]),        # Neuron indices
    types=np.array([...]),      # 0=E, 1=I neuron type (optional)
    populations=np.array([...]) # Population identifiers (optional)
)
```

### NetworkConfig

Complete network configuration with all biophysical parameters:

```python
from pinglab.types import NetworkConfig

config = NetworkConfig(
    # Population sizes
    N_E=800,                    # Excitatory neurons
    N_I=200,                    # Inhibitory neurons

    # Timing
    dt=0.1,                     # Time step (ms)
    T=5000.0,                   # Total duration (ms)

    # Synaptic delays (ms)
    delay_ei=1.0,               # E → I delay
    delay_ie=1.0,               # I → E delay
    delay_ee=1.0,               # E → E delay
    delay_ii=1.0,               # I → I delay

    # Coupling strengths
    g_ei=0.3,                   # E → I coupling
    g_ie=0.5,                   # I → E coupling
    g_ee=0.1,                   # E → E coupling
    g_ii=0.1,                   # I → I coupling

    # Connection probabilities
    p_ei=0.2,                   # E → I connection probability
    p_ie=0.2,                   # I → E connection probability
    p_ee=0.02,                  # E → E connection probability
    p_ii=0.02,                  # I → I connection probability

    # LIF neuron parameters
    E_L=-70.0,                  # Leak reversal potential (mV)
    E_e=0.0,                    # Excitatory reversal potential (mV)
    E_i=-80.0,                  # Inhibitory reversal potential (mV)
    C_m_E=200.0,                # E neuron membrane capacitance (pF)
    C_m_I=100.0,                # I neuron membrane capacitance (pF)
    g_L_E=10.0,                 # E neuron leak conductance (nS)
    g_L_I=10.0,                 # I neuron leak conductance (nS)
    V_th=-50.0,                 # Spike threshold (mV)
    V_reset=-60.0,              # Reset potential (mV)

    # Synaptic time constants
    tau_ampa=5.0,               # AMPA (excitatory) time constant (ms)
    tau_gaba=10.0,              # GABA (inhibitory) time constant (ms)

    # Refractory periods
    t_ref_E=2.0,                # E neuron refractory period (ms)
    t_ref_I=1.0,                # I neuron refractory period (ms)

    # Heterogeneity (standard deviations)
    V_th_heterogeneity_sd=0.0,  # Threshold variation (mV)
    g_L_heterogeneity_sd=0.0,   # Leak conductance variation (relative)
    C_m_heterogeneity_sd=0.0,   # Capacitance variation (relative)
    t_ref_heterogeneity_sd=0.0, # Refractory period variation (ms)

    # Connectivity scaling
    connectivity_scaling="one_over_N_src"  # or "none"
)
```

### InstrumentsConfig

Configuration for recording neural traces:

```python
from pinglab.types import InstrumentsConfig

instruments = InstrumentsConfig(
```

### InstrumentsResults

Recording output containing:
- `times` - Time points
- `neuron_ids` - Recorded neuron indices
- `types` - Neuron types (E/I)
- `V`, `g_e`, `g_i` - Individual neuron traces
- `V_mean_E`, `V_mean_I` - Population mean voltages
- `g_e_mean_E`, `g_e_mean_I` - Population mean excitatory conductances
- `g_i_mean_E`, `g_i_mean_I` - Population mean inhibitory conductances

Simulation output:

```python
from pinglab.types import NetworkResult
```

## Neuron Model

Pinglab implements a conductance-based leaky integrate-and-fire (LIF) neuron model. The membrane potential dynamics are governed by:

$$
C_m \frac{dV}{dt} = g_L(E_L - V) + g_e(E_e - V) + g_i(E_i - V) + I_{ext}
$$

Where:
- $C_m$ is the membrane capacitance
- $g_L$ is the leak conductance with reversal potential $E_L$
- $g_e$ is the excitatory (AMPA) conductance with reversal $E_e = 0$ mV
- $g_i$ is the inhibitory (GABA) conductance with reversal $E_i = -80$ mV
- $I_{ext}$ is external input current

When $V \geq V_{th}$, a spike is emitted, and $V$ is reset to $V_{reset}$ for a refractory period $t_{ref}$.

### Synaptic Dynamics

Synaptic conductances decay exponentially:

$$
\frac{dg}{dt} = -\frac{g}{\tau}
$$

With time constants $\tau_{AMPA}$ for excitatory and $\tau_{GABA}$ for inhibitory synapses.

## Running Simulations

### Basic Simulation

```python
from pinglab import run_network, inputs
from pinglab.types import NetworkConfig

# Create configuration
config = NetworkConfig(
    N_E=800, N_I=200,
    dt=0.1, T=5000.0,
    g_ei=0.3, g_ie=0.5,
    tau_ampa=5.0, tau_gaba=10.0
)

# Generate stimulus
I_ext = inputs.tonic(
    N_E=config.N_E,
    N_I=config.N_I,
    I_E=0.5,
    I_I=0.3,
    noise_std=0.1,
    num_steps=int(config.T / config.dt),
    seed=42
)

# Run simulation
result = run_network(config, I_ext)

# Access spikes
print(f"Total spikes: {len(result.spikes.times)}")
```

### With Instrument Recording

```python
from pinglab.types import InstrumentsConfig

config = NetworkConfig(
    N_E=800, N_I=200,
    dt=0.1, T=1000.0,
    instruments=InstrumentsConfig(
        variables=['V', 'g_e', 'g_i'],
        neuron_ids=[0, 100, 800],
        downsample=10,
        population_means=True
    )
)

result = run_network(config, I_ext)

# Access traces
V_traces = result.instruments.V           # Shape: (time, neurons)
V_mean_E = result.instruments.V_mean_E    # Shape: (time,)
```

### Stability Validation

The simulation engine automatically validates:
- **Numerical stability**: $dt < \tau_{min}/5$
- **Accuracy**: Warning if $dt > \tau_{min}/10$
- **Firing rates**: Sanity check against biological limits

## Input Generation

### Tonic Input

Constant baseline current with Gaussian noise:

```python
from pinglab import inputs

I_ext = inputs.tonic(
    N_E=800,
    N_I=200,
    I_E=0.5,          # Baseline current to E neurons
    I_I=0.3,          # Baseline current to I neurons
    noise_std=0.1,    # Noise standard deviation
    num_steps=50000,
    seed=42
)
# Shape: (num_steps, N_E + N_I)
```

### Oscillating Input

Sinusoidal modulation for studying oscillatory dynamics:

```python
I_ext = inputs.oscillating(
    N_E=800,
    N_I=200,
    I_E=0.5,
    I_I=0.3,
    amplitude=0.2,       # Oscillation amplitude
    frequency=40.0,      # Frequency in Hz
    phase_E=0.0,         # Phase for E population (radians)
    phase_I=np.pi/2,     # Phase for I population (radians)
    noise_std=0.1,
    num_steps=50000,
    dt=0.1,
    seed=42
)
```

Add step current pulses to existing input:

```python
I_ext_with_pulse = inputs.add_pulse_to_input(
```

## Analysis Functions

Pinglab provides comprehensive analysis tools organized by category.

### Firing Rate Analysis

```python
from pinglab import analysis

# Population mean firing rates (Hz)
rate_E, rate_I = analysis.mean_firing_rates(
    spikes, T=config.T, N_E=config.N_E, N_I=config.N_I
)

# Alternative interface
rate_E, rate_I = analysis.population_mean_rate(
    spikes, T=config.T, N_E=config.N_E, N_I=config.N_I
)

# Time-binned population rate
times, rate = analysis.population_rate(
    spikes, bin_width=10.0  # 10 ms bins
)
```

### Variability and Regularity

**Coefficient of Variation (CV) of ISI:**

The CV measures spike train regularity. For a Poisson process, $CV = 1$.

```python
cv_population, cv_per_neuron = analysis.population_isi_cv(spikes)
# cv_population: dict with 'E' and 'I' keys
# cv_per_neuron: dict with per-neuron CV values
```

**Fano Factor:**

The Fano factor is the variance-to-mean ratio of spike counts:

$$
FF = \frac{\text{Var}(N)}{\text{Mean}(N)}
$$

```python
ff_E, ff_I = analysis.population_fano_factor(
    spikes,
    T=config.T,
    window=50.0  # Counting window (ms)
)
```

### Synchrony Measures

**Synchrony Index:**

```python
sync_E, sync_I = analysis.synchrony_index(
    spikes,
    bin_width=5.0  # Binning for spike counts
)
```

**Pairwise Spike Count Correlation:**

```python
corr_EE, corr_II, corr_EI = analysis.pairwise_spike_count_corr(
    spikes,
    T=config.T,
    bin_width=50.0,
    n_pairs=100  # Number of pairs to sample
)
```

### Temporal Dynamics

**E→I Cross-Correlation:**

```python
lags, xcorr = analysis.crosscorr(
    spikes,
    max_lag=50.0,  # Maximum lag (ms)
    bin_width=1.0
)
```

**E→I Lag Statistics:**

```python
mean_lag, std_lag = analysis.ei_lag_stats(
    spikes,
    max_lag=20.0
)
```

### Spectral Analysis

**Power Spectral Density:**

```python
freqs, psd = analysis.rate_psd(
    spikes,
    T=config.T,
    fs=1000.0,        # Sampling frequency (Hz)
    nperseg=1024      # Welch segment length
)
```

**Gamma Band Metrics:**

```python
gamma_freq, gamma_power, gamma_q = analysis.gamma_metrics(
    spikes,
    T=config.T,
    fs=100.0,
    gamma_range=(30, 80)  # Gamma frequency band (Hz)
)
# gamma_freq: Peak frequency in gamma band
# gamma_power: Power at peak
# gamma_q: Quality factor (peak sharpness)
```

### Conductance Analysis

```python
g_stats = analysis.conductance_stats(result.instruments)
# Returns: mean_g_e, mean_g_i, cv_g_e, cv_g_i, ei_ratio
```

### Regime Classification

Classifies network operating state:

```python
regime = analysis.calculate_regime(
    rate_E=rate_E,
    rate_I=rate_I,
    cv_E=cv_E,
    cv_I=cv_I,
    gamma_power=gamma_power,
    sync_index=sync_E
)
# Returns: 'silent', 'AI', 'PING', 'burst', or 'unstable'
```

**Regime Definitions:**
- **Silent**: Very low firing rates
- **AI (Asynchronous Irregular)**: Low synchrony, high CV
- **PING**: Strong gamma oscillations, moderate synchrony
- **Burst**: High synchrony, irregular bursting
- **Unstable**: Extremely high firing rates

### Parameter Sweeps

Generate parameter grids for systematic exploration:

```python
from pinglab.utils import expand_parameter_spec

sweep_spec = {
    "g_ei": {"mode": "linspace", "start": 0.1, "stop": 1.0, "num": 10},
    "g_ie": {"mode": "linspace", "start": 0.1, "stop": 1.0, "num": 10},
    "tau_gaba": {"mode": "values", "values": [5.0, 10.0, 15.0]}
}

param_grid = expand_parameter_spec(sweep_spec)
# Returns list of dicts: [{"g_ei": 0.1, "g_ie": 0.1, "tau_gaba": 5.0}, ...]
```

**Sweep modes:**
- `linspace`: Linearly spaced values (start, stop, num)
- `range`: Integer range (start, stop, step)
- `values`: Explicit list of values

## Parallel Execution

Run multiple configurations in parallel using joblib:

```python
from pinglab.multiprocessing import parallel

def run_single(params):
    config = NetworkConfig(**{**base_params, **params})
    I_ext = inputs.tonic(...)
    return run_network(config, I_ext)

# Generate parameter grid
param_grid = expand_parameter_spec(sweep_spec)

# Run in parallel (uses all available cores)
results = parallel(run_single, param_grid)
```

## Complete Example

```python
import numpy as np
from pinglab import run_network, inputs, analysis
from pinglab.types import NetworkConfig, InstrumentsConfig
from pinglab.plots import save_raster, save_instrument_traces

# Configure network
config = NetworkConfig(
    N_E=800,
    N_I=200,
    dt=0.1,
    T=2000.0,
    g_ei=0.3,
    g_ie=0.5,
    g_ee=0.05,
    g_ii=0.1,
    tau_ampa=5.0,
    tau_gaba=10.0,
    V_th_heterogeneity_sd=2.0,
    instruments=InstrumentsConfig(
        variables=['V', 'g_e', 'g_i'],
        neuron_ids=[0, 400, 800, 900],
        downsample=10,
        population_means=True
    )
)

# Generate oscillating input at gamma frequency
I_ext = inputs.oscillating(
    N_E=config.N_E,
    N_I=config.N_I,
    I_E=0.5,
    I_I=0.4,
    amplitude=0.15,
    frequency=40.0,
    phase_E=0.0,
    phase_I=0.0,
    noise_std=0.05,
    num_steps=int(config.T / config.dt),
    dt=config.dt,
    seed=42
)

# Run simulation
result = run_network(config, I_ext)

# Analyze results
rate_E, rate_I = analysis.mean_firing_rates(
    result.spikes, config.T, config.N_E, config.N_I
)
print(f"Firing rates: E={rate_E:.1f} Hz, I={rate_I:.1f} Hz")

cv_pop, _ = analysis.population_isi_cv(result.spikes)
print(f"ISI CV: E={cv_pop['E']:.2f}, I={cv_pop['I']:.2f}")

gamma_freq, gamma_power, gamma_q = analysis.gamma_metrics(
    result.spikes, T=config.T, fs=100.0
)
print(f"Gamma: {gamma_freq:.1f} Hz, power={gamma_power:.2f}, Q={gamma_q:.2f}")

# Visualize
save_raster(result.spikes, "gamma_raster", dt=config.dt)
save_instrument_traces(result.instruments, "gamma_traces")
```

## Heterogeneity

Neural heterogeneity adds biological realism by introducing variability across neurons:

```python
config = NetworkConfig(
    # ... other parameters ...
    V_th_heterogeneity_sd=2.0,    # ±2 mV threshold variation
    g_L_heterogeneity_sd=0.1,     # ±10% leak conductance variation
    C_m_heterogeneity_sd=0.1,     # ±10% capacitance variation
    t_ref_heterogeneity_sd=0.5    # ±0.5 ms refractory variation
)
```

**Application methods:**
- `V_th`: Additive Gaussian ($V_{th,i} = V_{th} + \mathcal{N}(0, \sigma_{V_{th}})$)
- `g_L`, `C_m`: Multiplicative lognormal (preserves positivity)
- `t_ref`: Additive Gaussian with clipping to prevent negative values

## Connectivity Scaling

The `connectivity_scaling` parameter controls how synaptic weights scale with network size:

- **`"one_over_N_src"`**: Weights are normalized by source population size, maintaining constant total input as the network scales. This is the default and recommended for most applications.

- **`"none"`**: Raw weights are used directly. Useful for small networks or when specific weight values are required.

$$
g_{eff} = \begin{cases}
g / N_{src} & \text{if scaling = "one\_over\_N\_src"} \\
g & \text{if scaling = "none"}
\end{cases}
$$

## Best Practices

1. **Time step selection**: Use $dt \leq \tau_{min}/10$ for accurate results (typically $dt = 0.1$ ms)

2. **Warmup period**: Discard the first 500-1000 ms to allow transients to decay

3. **Heterogeneity**: Include moderate heterogeneity ($\sigma \approx 5-10\%$) for realistic dynamics

4. **Parallel sweeps**: Use `expand_parameter_spec` + `parallel` for efficient exploration

5. **Memory management**: For long simulations, use downsampling in `InstrumentsConfig`

6. **Reproducibility**: Always set random seeds for inputs and connectivity
