We want to test whether **oscillatory phase alignment** and **spike-time precision**
modulate effective communication into a receiver network
(“communication-through-coherence” Fries-style precision argument), without
confounds from a full sender network.

### Core idea

We don’t need a sender network. We use a **controlled spike-packet generator**
that can:

* place packets at specific **phases** of the receiver oscillation
* add controllable **timing jitter** (precision)

Then we measure how strongly the receiver responds.

## Setup

### Receiver network B

* A spiking network configured to show a clear oscillatory regime (e.g., PING
  gamma).
* Keep B in a *fixed operating point* throughout the experiment (don’t sweep
  g_ei during the main test).

**Outputs to record**

* Population rate or synaptic current proxy for phase estimation.
* Spike times of B (at least for E population).

### Input interface to B

* Create an “input fiber” population of size $N_\text{in}$.
* Each input fiber makes fixed excitatory synapses onto some subset of B’s
  excitatory neurons (sparse random projection is fine).
* Inputs are spikes only (no analog current required).

## Stimulus: spike packet generator

Define a packet by:

* **Packet time** $t_0$ (this is what we will shift to sweep phase)
* **Packet width** $w$ (e.g., 2–5 ms)
* **Packet size** $K$ (expected spikes per fiber or total spikes across fibers)
* **Jitter** $\sigma$ (Gaussian timing noise added to each spike time)

Implementation conceptually:

* For each packet event, for each input fiber:

  * generate 0/1 (or Poisson count) spikes
  * place spike times within $[t_0, t_0+w]$
  * add jitter: $t \leftarrow t + \mathcal{N}(0,\sigma^2)$

This creates a precisely controllable “message” whose timing precision we can
degrade smoothly.



## Phase definition and control

### Phase estimation (simple)

* Compute B's population rate $r_B(t)$ (or LFP proxy, e.g., mean synaptic
  current).
* Bandpass around the oscillation band (or just smooth if our oscillation is
  clean).
* Use Hilbert transform to get instantaneous phase $\phi_B(t)$.

### Phase targeting

Pick a reference phase (e.g., $\phi = 0$ at the peak of B's population rate). To
sweep phase, we have two easy methods:

1. **Delay sweep (recommended):**

* Keep packet schedule fixed in absolute time.
* Vary A→B effective delay $d$ applied to the packet spikes.
* This sweeps relative phase without changing packet generation logic.

2. **Time shift:**

* Keep synaptic delay fixed.
* Shift packet time $t_0$ relative to the estimated phase.

Delay sweep is usually simpler and more robust.



## Experimental manipulations

### 1) Phase sweep

* Choose a set of delays $d$ spanning one oscillation period $T$.

  * e.g., $d \in \{0, 2, 4, \dots, T\}$ ms.

### 2) Precision (jitter) sweep

* Choose jitter values $\sigma$ (ms), e.g. $\{0, 1, 2, 5\}$.
* Jitter is applied to transmitted spike times.

### Optional controls

* Keep packet “energy” constant across conditions (same expected total spike
  count per packet).
* If we’re worried about mean rate confounds, use baseline subtraction in the
  readout metric rather than trying to clamp rates aggressively.



## Readout metric (keep it brutally simple)

Pick one scalar response per packet:

### Recommended: evoked spike count

For each packet delivered at time $t_\text{arrive}$:

* count B excitatory spikes in a short window: $R = \#\text{spikes}_E \text{ in } [t_\text{arrive},\ t_\text{arrive}+\Delta]$
* baseline subtract using a pre-window: $R_\text{evoked} = R - \#\text{spikes}_E \text{ in } [t_\text{arrive}-\Delta,\ t_\text{arrive}]$

Use $\Delta \sim 10\text{–}20$ ms (big enough to capture the response, small enough not to
smear across phases).



## Expected result (the Fries signature)

Plot **mean evoked response** vs **phase (delay)** for each jitter level:

* At **low jitter**: response vs phase is **strongly modulated** (clear peak at
  “good phase”).
* As **jitter increases**: phase modulation **flattens** and overall evoked
  response drops.

That’s the precision argument in one figure: *timing precision determines how
effectively inputs exploit the oscillatory excitability window.*



## Minimal plotting checklist

1. B population rate trace + phase estimate (sanity)
2. Example raster around a few packets at "good phase" vs "bad phase"
3. Main figure: $\mathbb{E}[R_\text{evoked}]$ vs delay, with separate curves
   for $\sigma$



## How to expand this after the minimal test

### Expansion A: add a real sender network A

Replace the packet generator with an actual network A:

* A encodes a signal into spike packets (or population bursts).
* A→B coupling is the same projection.
* Repeat the same **delay** and **jitter** sweeps. This tests whether a
  realistic sender can exploit the same gating.

### Expansion B: two oscillatory networks + coherence

Make both A and B oscillatory (PING/PING):

* introduce a shared weak pacemaker or coupling so they can phase-lock
* measure how transfer depends on **phase difference** between their
  oscillations
* compare coherent vs incoherent conditions at matched mean rates

### Expansion C: “image sending”

Only after the above:

* encode image into multiple packet streams (patches or latent dims)
* measure reconstruction quality vs phase alignment + jitter
* keep the encoding fixed so we’re not re-learning a decoder while testing
  coherence

The key principle stays the same: **first demonstrate the phase+precision gate
with a scalar message**, then scale up bandwidth to images.
