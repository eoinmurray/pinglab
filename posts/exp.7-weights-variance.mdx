The goal of this post is to investigate how much variance in the synaptics
weights the network can handle and still be in a PING regime.

First we make some sanity plots of voltages vs and spikes vs time for MQIF
neurons:

Then we aim to make a heatmaps with the following properties:

1. x-axis: weight variance (stddev)
2. y-axis: mean weight
3. color: some measure of PING-ness

Our measure if PING-ness will be the Rhythmicity Index (RI) as used in the
[previous post on rhythmicity](./exp.5-rhythmicity-equation.mdx).

First a note on network connectivity: pinglab uses a mean-field style coupling
where each time step applies the expected conductance from population spike
counts to every target neuron, scaled by `p_ee`, `p_ei`, `p_ie`, `p_ii` and
(by default) `1/N_src`. There is no explicit adjacency matrix or per-connection
sampling.
