## Goal

Explore how mean and variance in synaptic weights affects the emergence PING rhythms.

We will do this by creating a heatmap of $\mu$ vs $\sigma$ vs PINGness metrics.

## Prior work

> Variance in the strength of I → E synapses has little effect on the resulting
> rhythm, while E → I variance reduces coherence by a factor of 0.5. In some
> cases, strong variance in both sets of synapses degrades the rhythm entirely.
> — Byrne 2024, p. 5.

Rory Byrne's report *"Stable and flexible computation in rhythmic spiking
networks"* found that variance in the $I \to E$ weights had little effect on the
PING rhythms, while variance in the $E \to I$ weights had a stronger effect.
Thus for our initial pass we will focus on $E \to I$ weights. In future we can
repeat the study on all weight types.

## Neuron Selection

For now we will use MQIF neurons, we have as of yet not done a full study of
which neuron type (LIF, MQIF, QIF, AdEx, HH) is best for PING rhythms. MQIF is a
reasonable starting point, given it’s proposed in Thomas Burgers thesis in the
context of PING networks.

## Rate matching

I believe we need to firing rate match all scans. This is because increasing
variance in weights can affect firing rates, which in turn affects oscillation
metrics. To isolate the effect of weight variance on oscillation quality, we
need to ensure firing rates are held constant.

First we run a short baseline simulation with the
default weights to measure the target $E$ firing rate. Then, for each $(\mu,
\sigma)$ pair in the scan, we do a coarse grid search over $I_E$ (holding $I_I$
fixed) to minimize the squared error between the observed $E$ rate and the
target. Finally we refine around the best $I_E$ with a tighter local search to
lock the rate in before running the full simulation for plots and metrics.

## Autocorrelation Rhythmicity Index (RI)

Definition as in [exp.3-rhythmicity-metric](./exp.3-rhythmicity-metric.mdx).

By taking the ratio of largest non-zero-lag peak (in gamma-lag window) to
zero-lag peak from the autocorrelation of the mean-subtracted population rate we
answer the question: *Does the rhythm repeat cleanly from cycle to cycle?*

**Strengths:**

* Captures temporal regularity.
* Detects stable oscillators vs noisy bursts.

**Weaknesses:**

* Sensitive to rate and silence if not normalized.
* Population-level only.

**Fails at catching:**

* Bursty, unstable “fake gamma”.

| Case | RI | Interpretation                        |
| ---- | -- | ------------------------------------- |
| 1    | ↑  | Stable, repeatable rhythm             |
| 2    | ↓  | Irregular or burst-driven oscillation |

What raster looks like in each case:

1. **↑ RI:** tight, repeating spike bands with clear cycle timing.
2. **↓ RI:** bursty or smeared bands with inconsistent timing.

## Pseudocode outline

```python
for mu in mu_values:
  for sigma in sigma_values:
    # Generate weights
    g_ei_weights = sample_weights(mu, sigma, size=(N_e, N_i))
    # Rate match external drive
    I_ext_e, I_ext_i = rate_match(g_ei_weights, target_rates)
    # Run simulation
    sim_data = run_simulation(g_ei_weights, I_ext_e, I_ext_i, other_params)
    # Compute metrics
    ri = compute_rhythmicity_index(sim_data)
    # Plot results
    plot_results(mu, sigma, ri, sim_data)
```
